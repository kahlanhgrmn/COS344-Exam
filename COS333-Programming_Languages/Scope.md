Examination Scope
I have discussed the examination scope in lectures. Here is a detailed breakdown of the material that is examinable, which includes all the material covered during the semester. This includes the scopes of Semester Tests 1 and 2.



Note that there will only be one implementation question in Scheme, and one implementation question in Prolog. Both will involve list processing, so be sure you can complete these kinds of implementations.



General



Note that the textbook covers material that the slides do not, and vice versa. Please study from both. Also note that I have included numerous notes for many of the slides, which can be viewed in PowerPoint or LibreOffice.



Also, make sure that you learn the terminology used throughout the textbook, because you must use the correct terms to discuss the programming language concepts. It is important to be as specific as possible when naming programming language features.



Chapter 1



The whole chapter is important, except for the discussion related to Figure 1.2, Section 1.7.4, and Section 1.8. In Section 1.7, you do not need to know the detailed steps of the compilation and hybrid implementation processes. For the material covering reasons for studying programming languages, I may give you a scenario related to a person (e.g. a junior programmer, a manager, a lecturer), and ask you for one or more reasons for them to study programming language concepts. For such questions, your answer must relate to the scenario I provide, meaning that you cannot simply select any reason. Make sure that you can apply the language evaluation criteria to features and design decisions in both real-world and hypothetical programming languages. Hypothetical programming languages are programming languages that do not necessarily exist, but for which I will describe language features. Pay special attention to the concept of orthogonality, which will definitely be tested.



Chapter 2



Leave out Sections 2.1, 2.2, 2.6.2, and 2.9.2. For Section 2.20, only focus on what's in the slides. For the remaining sections, don't focus on finer details. For each language, you need to know:



What the environment for its development and application is/was (this includes aspects such as the intended users, intended application area, software engineering approaches that were popular at the time of development, and general characteristics of the computer system for which the language was designed).
How the environment you have identified affected the language.
The main contribution (or contributions) of each language.


You don't need to know specific features that were introduced in certain versions of the language, dates, specific model names and model numbers of the computers the languages were designed for, the names of the programming language developers, or details about the design committees.



Chapter 15



This chapter is very important. Study all the material up to and including Section 15.5, and Section 15.5.11. In other words, cover all the material up to and including the Scheme examples of list processing functions. Also, don't study Common Lisp, ML, Haskell, F#, and support for functional programming in imperative programming languages). Leave out Sections 15.2 and 15.4 (reading these sections may consolidate concepts in the chapter, but are not directly examinable). Note that the slides contain material covered in Sections 7.2.2, 8.2.2.4, 6.9, and 5.5.2, which is examinable. It recommended that students study the chapter using both the slides and textbook. The lambda expressions in Section 15.5.4 are also not examinable.



You need to be able to analyse and write simple Scheme programs no more complex than the example covered in class (the member, equalsimp, equal, and append functions) and the practical. I VERY STRONGLY recommend that all students complete Practical 2, because it provides good practice for the program code implementation questions you can expect. There will be one question that requires you to write Scheme code, for which you will have to use the DrRacket interpreter (as described in the practical specification) under Windows. Therefore, if you do your practical using a different interpreter (such as an online interpreter), make sure you familiarise yourself with the DrRacket interpreter before the test.



There are several additional examples of Scheme implementation questions, which students can use as exercises, in the example test and examination papers.



Note that I will only ask one Scheme implementation question involving list processing.



Chapter 16



This chapter is very important. All material in the chapter is examinable. You need to be able to analyse and write simple Prolog programs no more complex than the examples covered in class (the member, append, and reverse propositions) and the practicals. I VERY STRONGLY recommend that all students complete Practicals 3 and 4, because they provide good practice for the program code implementation questions you can expect.



There will be one question that requires you to write Prolog code, for which you will have to use the SWI-Prolog interpreter (as described in the Practical 3 specification) under Windows. Therefore, if you implemented your Prolog practical tasks using a different interpreter (such as an online interpreter), make sure you familiarise yourself with the SWI-Prolog interpreter before the test. Also note that the same restrictions on which elements of the Prolog language you are allowed to use, which are laid out in the specifications for Practicals 3 and 4, will also apply to the test questions. You may only use the language features described in the textbook and the additional hints provided in the practical specifications.



There are several additional examples of Prolog implementation questions, which can be used as extra exercises, in the example test and examination papers.



During the test you will have access to the SWI_Prolog documentation and the specifications of Practicals 3 and 4. Note that these are meant only for quick referencing. You should not be referring heavily to any of these documents. This therefore means that you should still thoroughly study Chapter 16, and be able to write Prolog programs largely without assistance.



Note that I will only ask one Prolog implementation question involving list processing.
