Ch9
Subprograms
Must know: all parameter passing methods (in-mode, out-mode, in-out-mode, pass-by-value, pass-by-result, pass-by-value-result, pass-by-reference, pass-by-name) · referencing environments · generic subprograms in C++/Java/C# · closures in JavaScript (MakeAdder pattern — can be asked for output) · coroutines (be able to trace output from pseudocode) · Sections 9.6 & 9.7 are critical.
~6 marks
Ch9 Steps
  • Read the Ch9 cheatsheet — scan all terminology before opening slides Cheatsheet →
  • Study slides + textbook: parameter passing (9.5), referencing environments (9.5.3 excluded), generic subprograms 9.10.1–9.10.3, closures 9.12, coroutines 9.13
  • Do all Ch9 MCQs — note every wrong answer and re-read that section MCQ Bank →
  • Trace through 2–3 closure examples (MakeAdder style) and write the output by hand before checking
  • Trace through a coroutine pseudocode example — track which coroutine is active at each yield/resume
Ch12
Object-Oriented Programming
Must know: all OOP concepts (encapsulation, inheritance, polymorphism, dynamic dispatch/dynamic binding) · abstract classes vs interfaces · single vs multiple inheritance · Sections 12.5 and 12.6 are NOT examinable. Study Ch11 only if you need to clarify a term.
~5 marks
Ch12 Steps
  • Read the Ch12 cheatsheet — map out the OOP concept tree in your head Cheatsheet →
  • Study slides + textbook Ch12 (all except 12.5, 12.6) — pay attention to dynamic binding and how languages implement it differently
  • Do all Ch12 MCQs MCQ Bank →

Exclude: Figure 1.2, Section 1.7.4, Section 1.8.

Ch1 Steps

Ch15 Code Q
Functional Programming — Scheme
Must know: all material up to and including Section 15.5.11 (Scheme list processing) · functional forms · referential transparency & side effects (from Ch7/15 slides) · also note slides include material from sections 7.2.2, 8.2.2.4, 6.9, 5.5.2. Exclude: Common Lisp, ML, Haskell, F#, Section 15.2, 15.4, lambda expressions 15.5.4. One implementation question in DrRacket.

Ch15 Steps

  • Study Ch15 slides + textbook up to 15.5.11 — understand car/cdr/cons, let/letrec, define, cond

Logic Programming — Prolog

Ch2
Programming Language History
Must know: for each language — (1) environment/intended users/application area, (2) how the environment affected the language, (3) main contributions. Focus on what's in the slides (Section 2.20: slides only). Exclude: 2.1, 2.2, 2.6.2, 2.9.2. Do not memorise dates, model numbers, or developer names.
~1 mark
Ch2 Steps
  • Read the Ch2 cheatsheet — this is a quick chapter, focus on the 3-point framework for each language Cheatsheet →
  • Scan Ch2 slides — for each language, write 1 sentence per framework point (environment → effect → contribution)
  • Do all Ch2 MCQs MCQ Bank →
