Part 1 — Scheme

5 marks
What the exam asks

One 5-mark Scheme question per exam. You get a function name and a plain-English description of what to do to a simple numeric list. Write a complete recursive Scheme function that yields (returns) the result — never prints it.

Key restriction

positive? is NOT in the allowed list — use (> x 0) instead. Everything else you expect is allowed: negative?, zero?, even?, odd?, null?, abs, car, cdr, cons, cond, let.

The 4 Patterns

A
Filter — keep or remove elements
Output list has ≤ input length. Clue words: "remove all X", "keep only X", "without X".
Scheme Template
(define (functionName lst)
  (cond
    ((null? lst) '())
    ((KEEP-CONDITION (car lst))
     (cons (car lst)
           (functionName (cdr lst))))
    (else
     (functionName (cdr lst)))
  )
)
Prolog Template
myProp([], []).

myProp([H|T], [H|R]) :-
    H > 0,
    myProp(T, R).

myProp([H|T], R) :-
    H =< 0,
    myProp(T, R).
Rule

If condition true → cons the element and recurse. If false → just recurse (skip element).

B
Map — transform every element
Output length = input length. Every element appears, possibly changed. Clue words: "transform every element", "double all", "square each".
Scheme Template
(define (functionName lst)
  (cond
    ((null? lst) '())
    (else
     (cons (TRANSFORM (car lst))
           (functionName (cdr lst))))
  )
)
Prolog Template
myProp([], []).

myProp([H|T], [New|R]) :-
    New is EXPRESSION,
    myProp(T, R).
Rule

No element is ever skipped. Only 2 clauses (Prolog) / 2 cond-cases (Scheme): base + single recursive.

C
Filter+Map — 3-way split (exam favourite)
Some elements omitted, some kept, some transformed. This is what the 2025 exam used. Clue: "X unchanged, Y transformed, Z omitted".
Scheme Template
(define (functionName lst)
  (cond
    ((null? lst) '())
    ((OMIT-CONDITION (car lst))
     (functionName (cdr lst)))
    ((TRANSFORM-CONDITION (car lst))
     (cons (TRANSFORM (car lst))
           (functionName (cdr lst))))
    (else
     (cons (car lst)
           (functionName (cdr lst))))
  )
)
Prolog Template
myProp([], []).

% omit this element
myProp([H|T], R) :-
    H =:= 0,
    myProp(T, R).

% transform this element
myProp([H|T], [New|R]) :-
    H > 0,
    New is H * 2,
    myProp(T, R).

% keep unchanged
myProp([H|T], [H|R]) :-
    H < 0,
    myProp(T, R).
Critical

Make conditions mutually exclusive — every element must match exactly one clause/branch. Use <0, =:=0, >0 (not ≥0 with >0) to avoid overlaps.

D
Accumulate — return a number (count or sum)
Returns a single number, not a list. Base case is 0. Clue words: "return the count of", "return the sum of".
Scheme Template
(define (functionName lst)
  (cond
    ((null? lst) 0)
    ((CONDITION (car lst))
     (+ CONTRIBUTION
        (functionName (cdr lst))))
    (else
     (functionName (cdr lst)))
  )
)
Prolog Template
myProp([], 0).

myProp([H|T], N) :-
    H > 0,
    myProp(T, Rest),
    N is Rest + 1.

myProp([H|T], N) :-
    H =< 0,
    myProp(T, N).
Rule

Base case returns 0 (not '()). For count: add 1. For sum: add the element value. Non-matching elements contribute nothing.

Step-by-Step: Scheme

Exam technique
  1. Identify the pattern from the description: "remove/keep" → A · "transform every" → B · "3 categories" → C · "return count/sum" → D
  2. Write the base case first: (null? lst) → '() for A/B/C, 0 for D
  3. Write condition cases using cond (prefer cond over nested if for 3+ cases)
  4. Use cons to build the output list — never wrap with list
  5. Double-check:
    • Is positive? needed? Use (> x 0) instead
    • Is the function named exactly as stated?
    • Does it yield, not print? No display calls
    • Is the empty list quoted? '() not ()

Part 2 — Prolog

5 marks each
What the exam asks

Every exam has one 5-mark Prolog question; ST2 has two. Always a list-processing proposition name(L1, L2) where L2 is L1 with some filter/map applied.

Hard restrictions

No if-then, no if-then-else, no ; outside lists, no built-in member/append/reverse. Use only: constants, variables, list patterns, arithmetic, relational expressions, is, cuts, and built-in not.

Prolog List Proposition Structure

Every list proposition follows this structure:

% Base case: empty list
myProp([], []).

% Recursive cases: one clause per element outcome
myProp([Head|Tail], Result) :-
    % conditions on Head,
    % compute transformed head,
    % recurse on tail
  • The second parameter Result is your "return value" — it gets built up
  • Each case is a separate clause (not nested if-then)
  • Always put the base case [] first
  • Use [Head|Tail] to decompose, [Head|Result] or [NewHead|Result] to construct

Result patterns per element outcome:

Outcome for element HProlog result parameter
Element is omittedR (result = just the tail's result)
Element is kept unchanged[H|R]
Element is transformedCompute New is ..., then [New|R]

Step-by-Step: Prolog

  1. Write the base case first: myProp([], []).
  2. Count the cases for each element: how many different outcomes? 2 → Pattern A/B, 3 → Pattern C, counting/summing → Pattern D
  3. Write one clause per case:
    • Omit: myProp([H|T], R) :- Condition, myProp(T, R).
    • Keep: myProp([H|T], [H|R]) :- Condition, myProp(T, R).
    • Transform: myProp([H|T], [New|R]) :- Condition, New is Expr, myProp(T, R).
  4. Make conditions mutually exclusive: use H > 0, H < 0, H =:= 0
  5. Double-check:
    • Use =:= for numeric equality (not =)
    • Use is for arithmetic assignment
    • No ; operators or if-then constructs
    • Base case first

Common Mistakes

Scheme
✗
Returning () without quote → must be '()
✗
Using positive? → not allowed; use (> x 0)
✗
Printing instead of yielding → no display
✗
Using (list ...) to build output → use cons
✗
Forgetting the quote in base case entirely
Prolog
✗
Using = for numeric equality → use =:=
✗
Using X = expr for arithmetic → use X is expr
✗
Using ; for logical or → write separate clauses
✗
Using if-then (cond -> then ; else) → separate clauses
✗
Overlapping conditions → backtracking gives multiple answers
