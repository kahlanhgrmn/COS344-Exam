COS344 — GLSL Shader Coding Reference

1. GLSL Basics

Version & Profile


#version 330 core

Always the first line. 330 = OpenGL 3.3. core = no deprecated features.

Built-in Variables

Data Types

gl_PositionVertexMust set. Clip-space position
gl_PointSizeVertexSize of point primitives
gl_FragCoordFragmentWindow-space (x,y,z,1/w) of fragment
gl_FrontFacingFragmentTrue if front face
gl_FragDepthFragmentOverride depth (default = gl_FragCoord.z)

Constructors & Swizzling


vec4 a = vec4(1.0, 0.5, 0.0, 1.0);
vec3 b = a.rgb;           // (1.0, 0.5, 0.0)
vec2 c = a.xy;            // (1.0, 0.5)
float d = a.z;            // 0.0
vec4 e = vec4(b, 1.0);    // extend vec3 to vec4
vec3 f = vec3(0.5);       // all components = 0.5

mat4 identity = mat4(1.0);         // 4×4 identity matrix
mat4 M = mat4(col0, col1, col2, col3);  // column-major!

Exam trap: GLSL 1 is an integer literal. Float operations need 1.0. Mixing causes type errors.

2. Vertex Shader — Full Annotated Template


#version 330 core

// Input attributes — match glVertexAttribPointer locations
layout(location = 0) in vec3 aPos;       // vertex position in object space
layout(location = 1) in vec3 aNormal;    // vertex normal in object space
layout(location = 2) in vec2 aTexCoord; // UV coordinates

// Uniforms — same value for every vertex in a draw call
uniform mat4 model;       // object → world space
uniform mat4 view;        // world → camera space
uniform mat4 projection;  // camera → clip space

// Outputs — interpolated and passed to fragment shader
out vec3 FragPos;    // world-space position (for lighting)
out vec3 Normal;     // world-space normal (for lighting)
out vec2 TexCoord;  // UV coords (for texture sampling)

void main() {
    // World-space position (for lighting calculations)
    FragPos = vec3(model * vec4(aPos, 1.0));

    // Normal matrix: transpose(inverse(model)) — corrects non-uniform scaling
    // NOTE: mat3() extracts the upper-left 3×3 (no translation)
    Normal = mat3(transpose(inverse(model))) * aNormal;

    // Pass UV through unchanged
    TexCoord = aTexCoord;

    // Final clip-space position (OpenGL uses this for rasterization)
    gl_Position = projection * view * vec4(FragPos, 1.0);
}

Why transpose(inverse(model)) for normals?

If the model matrix has non-uniform scaling, naively multiplying normals by the model matrix will make them no longer perpendicular to the surface. The normal matrix = transpose of the inverse of the model matrix, which correctly transforms normals.

Why vec4(aPos, 1.0) and not vec4(aPos, 0.0)?

  • w = 1.0 → point (translation applies)
  • w = 0.0 → direction/vector (translation doesn't apply)

Positions are points; normals/directions use w=0.

3. Fragment Shader — Phong Lighting (Fully Annotated)


#version 330 core

// Received from vertex shader (interpolated across triangle)
in vec3 FragPos;
in vec3 Normal;
in vec2 TexCoord;

// Light and camera properties
uniform vec3 lightPos;    // world-space light position
uniform vec3 viewPos;     // world-space camera position
uniform vec3 lightColor;  // RGB color of light
uniform vec3 objectColor; // base surface color

// Output
out vec4 FragColor;

void main() {
    // ── AMBIENT ──────────────────────────────────────────
    // Constant low-level light simulating indirect illumination
    float ambientStrength = 0.1;
    vec3 ambient = ambientStrength * lightColor;

    // ── DIFFUSE (Lambert's Law) ───────────────────────────
    vec3 norm     = normalize(Normal);                  // re-normalize after interpolation
    vec3 lightDir = normalize(lightPos - FragPos);      // direction from surface to light

    // dot(n,l): cosine of angle between normal and light direction
    // max(..., 0): clamp negative values (light behind surface = no contribution)
    float diff    = max(dot(norm, lightDir), 0.0);
    vec3  diffuse = diff * lightColor;

    // ── SPECULAR (Blinn-Phong) ────────────────────────────
    float specularStrength = 0.5;
    vec3 viewDir = normalize(viewPos - FragPos);        // direction to camera

    // Blinn-Phong: use half-vector h = normalize(l + v)
    // Half-vector is between light direction and view direction
    vec3  halfVec  = normalize(lightDir + viewDir);
    float spec     = pow(max(dot(norm, halfVec), 0.0), 32.0);
    //                                              ^^^^
    //                              shininess: higher = tighter, smaller highlight
    vec3  specular = specularStrength * spec * lightColor;

    // ── COMBINE ──────────────────────────────────────────
    vec3 result = (ambient + diffuse + specular) * objectColor;
    FragColor = vec4(result, 1.0);
}

Phong specular (classic):


vec3 reflectDir = reflect(-lightDir, norm);  // reflect across normal
float spec = pow(max(dot(viewDir, reflectDir), 0.0), 32.0);

Why max(dot(...), 0.0)?

If the angle between normal and light direction > 90°, the dot product is negative — meaning the surface faces away from the light. max(..., 0) clamps this to zero (no contribution from lights behind the surface).

Shininess Exponent


α = 2:    very wide, matte-like highlight
α = 8:    broad highlight (rough surface)
α = 32:   medium highlight (plastic)
α = 128:  tight highlight (polished metal)
α = 256+: mirror-like

4. Coordinate Space Pipeline


Object Space
    │  × Model matrix M (rotation, scale, translation)
    ▼
World Space
    │  × View matrix V (camera look-at)
    ▼
View/Camera Space (camera at origin, looking down -Z)
    │  × Projection matrix P
    ▼
Clip Space (w ≠ 1 for perspective)
    │  ÷ w  (perspective divide — done automatically by OpenGL)
    ▼
NDC  [-1, 1]³
    │  × Viewport matrix Mvp
    ▼
Screen/Window Space  [0, width] × [0, height]

The MVP Matrices


gl_Position = projection * view * model * vec4(vertex, 1.0);
//  P            V          M

Matrix composition order (read right-to-left = applied left-to-right):

  1. Model M: place object in world
  2. View V: move world so camera is at origin
  3. Projection P: flatten 3D → 2D with depth

Model Matrix

Combines translate × rotate × scale (applied right-to-left):


glm::mat4 model = glm::translate(glm::mat4(1.0f), position)
                * glm::rotate(glm::mat4(1.0f), angle, axis)
                * glm::scale(glm::mat4(1.0f), scale);

View Matrix (Look-At)


glm::mat4 view = glm::lookAt(
    cameraPos,   // eye position
    cameraPos + cameraFront,  // target point
    cameraUp     // up direction (usually 0,1,0)
);

Projection Matrices

Perspective:


glm::mat4 proj = glm::perspective(
    glm::radians(fov),  // vertical FOV
    aspectRatio,        // width/height
    nearPlane,          // 0.1f
    farPlane            // 100.0f
);

Orthographic:


glm::mat4 proj = glm::ortho(left, right, bottom, top, near, far);

5. Texture Sampling in Shaders

Vertex Shader (pass UV through)


layout(location = 2) in vec2 aTexCoord;
out vec2 TexCoord;
void main() {
    TexCoord = aTexCoord;
    // ...
}

Fragment Shader (sample texture)


uniform sampler2D texture1;
in vec2 TexCoord;

void main() {
    vec4 texColor = texture(texture1, TexCoord);  // GLSL 330+
    // texture2D() is deprecated but equivalent

    // Combine with lighting:
    vec3 result = lighting * vec3(texColor);
    FragColor = vec4(result, texColor.a);
}

Multiple Textures


uniform sampler2D diffuseMap;
uniform sampler2D specularMap;

void main() {
    vec3 diffuseTex  = vec3(texture(diffuseMap, TexCoord));
    float specularTex = texture(specularMap, TexCoord).r;  // grayscale
    // Use specularTex to scale specular strength per-pixel
}

UV coordinates: (0,0) = bottom-left, (1,1) = top-right. Values outside [0,1] use wrapping mode (repeat/clamp).

6. Passing Data Between Shader Stages

The in/out System

Vertex shader output → fragment shader input must have matching names and types:


// Vertex shader
out vec3 FragPos;    // output named "FragPos"
out vec3 Normal;
out vec2 TexCoord;

// Fragment shader
in vec3 FragPos;     // input with SAME name "FragPos" — they connect
in vec3 Normal;
in vec2 TexCoord;

Interpolation Qualifiers

PhongBlinn-Phong
Specular vectorReflection r = reflect(-l, n)Half-vector h = normalize(l+v)
Formuladot(r, v)^αdot(n, h)^α
PerformanceSlower (reflect is expensive)Faster
AccuracyMore physically accurateGood approximation
Shininess for same lookα~4α

Uniforms

Set once from CPU, same value for every vertex/fragment in a draw call:


uniform mat4 model;        // set per-object
uniform vec3 lightPos;     // set once per frame
uniform sampler2D texture1; // texture unit binding

Exam trap: Uniforms are read-only in GLSL. You cannot write to them from a shader.

7. Common Shader Patterns

Gamma Correction

Monitor gamma ≈ 2.2; linear-space rendering → apply gamma before output:


FragColor.rgb = pow(fragColor.rgb, vec3(1.0 / 2.2));

Alpha Discard (for foliage, fences, etc.)


vec4 texColor = texture(texture1, TexCoord);
if (texColor.a < 0.1) discard;  // punch-out transparency
FragColor = texColor;

discard exits the fragment shader without writing to the framebuffer.

Fog Effect


float dist    = length(viewPos - FragPos);
float fogFactor = exp(-fogDensity * dist);          // exponential fog
fragColor     = mix(fogColor, fragColor, fogFactor); // blend toward fog color

Normal Mapping (Tangent Space)


// Sample normal map (values in [0,1], remap to [-1,1])
vec3 normalMap = texture(normalMapTex, TexCoord).rgb;
normalMap = normalMap * 2.0 - 1.0;  // remap to [-1,1]
// Transform from tangent space to world space using TBN matrix
vec3 N = normalize(TBN * normalMap);

Environment/Reflection Map


uniform samplerCube skybox;
vec3 viewDir  = normalize(FragPos - cameraPos);
vec3 reflectDir = reflect(viewDir, normalize(Normal));
FragColor     = texture(skybox, reflectDir);

Fresnel Effect (simplified Schlick)


float fresnel = R0 + (1.0 - R0) * pow(1.0 - max(dot(Normal, viewDir), 0.0), 5.0);
// Mix between refraction and reflection based on fresnel

8. Exam-Ready Shader Checklist

Before writing a shader in an exam:

Vertex Shader

  • [ ] #version 330 core on first line
  • [ ] All in attributes have layout(location = N)
  • [ ] gl_Position is set (in clip space: P V M * vec4(pos, 1.0))
  • [ ] w component is 1.0 for positions, 0.0 for directions
  • [ ] All outputs declared with out with correct names for fragment shader
  • [ ] Normal matrix used if non-uniform scaling possible

Fragment Shader

  • [ ] #version 330 core on first line
  • [ ] All in variables match vertex shader out names exactly
  • [ ] out vec4 FragColor declared
  • [ ] FragColor is set (all paths, or use discard)
  • [ ] normalize(Normal) before dot products (interpolation un-normalizes)
  • [ ] max(dot(...), 0.0) on all lighting dot products
  • [ ] Correct matrix multiplication order in vertex shader: P × V × M × pos
