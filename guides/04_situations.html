Exam Strategy — Situation Questions

⚠ What the exam asks

You are given a scene description and must justify every technical decision. For each choice state: the technique, why it fits this scene, and what you would not use instead.

How to Answer Any Scenario

  1. Outdoor or indoor/urban? → frustum culling (outdoor) vs BSP tree (indoor/urban)
  2. How many lights? → ≤10 lights: Forward rendering   |   >10 lights: Deferred rendering
  3. Any metal, glass, or water? → PBR + Fresnel; glass/water also needs Snell’s law
  4. Any transparency? → glass/water: alpha blend sorted back→front   |   foliage: alpha discard
  5. Any characters or creatures? → skeletal animation + smooth skinning; IK for foot placement
  6. Particles, weather, or fluids? → particle system (velocity, lifespan, spawn rate)
  7. Any glow or post-effects? → emissive material + bloom; volumetric for light shafts

Decision Tables

Lighting & Shadows

If the scene has…UseNot
Single sun / clear skyDirectional light + shadow mapsRay-traced shadows (too slow)
Soft natural outdoor shadowsShadow maps + PCF (average nearby samples)Hard shadow maps (look unnatural)
Many point lights (>10)Deferred rendering — G-buffer separates geometry from lightingForward rendering (cost = fragments × lights)
Glowing objects (fire, neon, bioluminescence)Emissive material + bloom post-processA point light per glow (too many lights)
No clear light direction (deep water, cell interior)Ambient lighting onlyDirectional shadows (no defined angle)

Materials & Surface Detail

If the scene has…UseNot
Metal surfaces (brass, iron, chrome)PBR metallic/roughness + Fresnel: R0+(1−R0)(1−cosθ)5Phong (no metallic/dielectric distinction)
Stone, cloth, woodBlinn-Phong (diffuse + specular sufficient)Full PBR (overkill for non-metals)
Glass / water / iceFresnel for reflection ratio + Snell’s law for refraction: n1sinθ1 = n2sinθ2Simple flat reflection
Bumpy surfaces (stone, scales, rust)Normal maps (per-pixel shading, no extra geometry)Adding actual geometry (polygon count explodes)
Terrain / snow / cobblestoneDisplacement map (moves actual vertices → correct silhouette)Normal maps alone (no silhouette bump)
Organic material (skin, membrane)Subsurface scattering (SSS) — light scatters through thin layerOpaque diffuse (loses translucent glow)

Transparency

If the scene has…UseNot
Glass, water, potions, hologramsAlpha blending sorted back→front: c = αcf + (1−α)cbZ-buffer only (wrong compositing order)
Foliage, fences, cutout shapesAlpha discard (discard in shader) — no sorting neededAlpha blending (sorting cost for thousands of leaves)
Two-sided transparent objects (bottles, membranes)Disable backface cullingBackface culling (hides inner surface)
Additive glow (magic, neon)Additive blending: GL_ONE, GL_ONE — colours only add, never darkenStandard alpha blend (would darken surroundings)

Animation & Motion

If the scene has…UseNot
Human / creature charactersSkeletal animation (FK) — transforms propagate parent → childRigid per-mesh animation (candy-wrapper artefact)
Character skin deformationSmooth skinning: d = ∑ωidi  (∑ωi = 1)Rigid skinning (sharp crease at every joint)
Foot / hand placement on uneven terrainInverse kinematics (IK) — endpoint-driven, adapts to surfaceFK only (feet float or clip through ground)
Mechanical rotation, floating objectsKeyframe / procedural rotation matrixFull skeletal rig (unnecessary complexity)
Rain, fire, dust, smoke, steamParticle system (velocity, lifespan, spawn rate, randomness)Static meshes (no dynamic per-particle motion)

Culling & Optimisation

If the scene has…UseNot
Open outdoor landscapeFrustum culling — discard geometry outside view volume before drawDrawing everything regardless
Dense urban / indoor roomsBSP tree — precomputed visibility, buildings occlude each otherFrustum culling alone (misses occlusion)
Dense complex geometry (jungle, coral, city)BVH — hierarchical bounding volumes, O(log n) intersectionLinear mesh tests O(n)
Many identical objects (crowd, fleet, trees)Instanced rendering — one draw call for N copiesN separate draw calls (CPU bottleneck)
Distant objects in large sceneLOD — swap high-poly for low-poly at distance thresholdFull detail at every distance

Rendering Pipeline

If the scene has…UseNot
Few lights, simple sceneForward rendering — lower overhead, straightforwardDeferred (G-buffer cost not worth it)
Many dynamic point lightsDeferred rendering — G-buffer separates geometry & lighting passesForward (performance collapses)
Transparent objects + many lightsDeferred for opaque + forward pass for transparent / particlesPure deferred (alpha blending incompatible with G-buffer)
Reflective metal / polished surfacesEnvironment map (cube map) — precomputed, cheap lookupReal-time ray tracing (too expensive)
Light scattering through medium (water, fog)Volumetric lighting (ray marching or texture slices)Plain ambient (no directional scatter)
Scoring tip

Always give a reason: “Shadow maps because there is a single directional light and they are efficient for real-time” scores higher than just naming the technique. State what you rejected and why.
