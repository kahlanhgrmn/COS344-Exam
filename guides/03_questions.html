L01 — Introduction to Computer Graphics (Ch1)

MCQs

1. What is computer graphics primarily concerned with?

A) Converting analogue signals to digital

B) Creating and manipulating visual content using computers

C) Compressing video files for storage

D) Designing microchip architectures

Answer: B — Computer graphics is the field of creating, rendering, and manipulating visual content through computational methods.

2. Which of the following is NOT a typical application of computer graphics?

A) Medical imaging

B) Video game rendering

C) Sorting algorithms

D) Film visual effects

Answer: C — Sorting algorithms are a data structures/algorithms topic, not computer graphics.

3. What is the difference between object-order and image-order rendering?

A) Object-order processes each pixel first; image-order processes each object first

B) Object-order processes each object first finding affected pixels; image-order processes each pixel first finding influencing objects

C) They are identical in approach

D) Object-order only works in 2D; image-order only in 3D

Answer: B — Object-order (rasterisation) iterates over objects; image-order (ray tracing) iterates over pixels.

4. Which rendering approach does ray tracing follow?

A) Object-order rendering

B) Image-order rendering

C) Hybrid rendering

D) Forward rendering

Answer: B — Ray tracing considers each pixel in turn and finds all objects that influence it.

5. A graphics application needs to render a scene where you must calculate the exact contribution of every light source per pixel for global illumination. Which rendering paradigm is more naturally suited?

A) Object-order (rasterisation)

B) Image-order (ray tracing)

C) Both are equally suited

D) Neither; use a lookup table

Answer: B — Image-order rendering (ray tracing) naturally traces light paths per pixel, making global illumination more straightforward.

6. Which of the following fields does NOT directly use computer graphics?

A) Architecture

B) Medicine

C) Cryptography

D) Automotive design

Answer: C — Cryptography deals with secure communication and does not inherently require computer graphics.

7. In a graphics pipeline, what role does the "shader" play?

A) Manages file storage on the GPU

B) A programmable stage that computes colour and geometry transformations

C) Compresses geometry for network transmission

D) Only handles shadow computation

Answer: B — Shaders are small programmable programs running on the GPU that control vertex transformations and fragment colour output.

8. Why is the GPU preferable to the CPU for rendering tasks?

A) GPUs have a faster clock speed than CPUs

B) GPUs support massive parallelism, processing thousands of threads simultaneously

C) GPUs use less power

D) CPUs cannot perform floating-point arithmetic

Answer: B — GPUs are designed with thousands of smaller cores optimised for parallel computation, ideal for per-vertex and per-pixel operations.

9. Which statement best describes the relationship between ray tracing and rasterisation in terms of performance vs quality?

A) Rasterisation is higher quality and faster

B) Ray tracing is faster but lower quality

C) Ray tracing produces more physically accurate results but is computationally expensive; rasterisation is faster but requires approximations

D) They always produce identical results

Answer: C — Ray tracing simulates light transport more accurately but is expensive; rasterisation uses tricks (shadow maps, ambient occlusion) to approximate.

10. A game developer needs to render a scene with 100,000 triangles at 60 fps in real time. Which rendering method is most appropriate and why?

A) Ray tracing; it visits each pixel only once

B) Rasterisation (object-order); it visits each triangle once and is highly parallelisable on GPUs

C) Path tracing; it handles all lighting automatically

D) Scanline rendering with CPU only

Answer: B — Rasterisation is designed for real-time GPU rendering, visiting each object once and leveraging massive parallelism.

Long Questions

Q1. Compare object-order rendering and image-order rendering. Describe the algorithmic approach of each and explain one scenario where each is preferred.

Model Answer:

  • Object-order (rasterisation): iterates over each geometric primitive, determines which pixels it covers, and updates those pixels — efficient for real-time.
  • Image-order (ray tracing): iterates over each pixel, casts a ray, finds intersecting objects, and shades — naturally handles reflections and global illumination.
  • Object-order is preferred for real-time applications (games) where speed matters and lighting approximations are acceptable.
  • Image-order is preferred for offline rendering (films, arch-viz) where physical accuracy and effects like caustics, refractions, and global illumination are needed.
  • The core trade-off is performance vs physical correctness.

Q2. A medical imaging company wants to visualise a 3D scan of a patient's brain interactively and also produce photorealistic still images for reports. Which rendering approach would you recommend for each use case, and why?

Model Answer:

  • Interactive visualisation: use rasterisation (object-order), as it renders at real-time frame rates allowing the doctor to rotate and inspect the model.
  • Photorealistic stills: use ray tracing (image-order), as it accurately simulates light scattering through tissue, producing physically correct highlights and depth.
  • Rasterisation approximates lighting with techniques like Phong shading; ray tracing computes actual light paths.
  • This dual approach is common in professional tools: fast interactive preview + high-quality offline render.
  • The key factor is the time budget per frame.

Q3. If a screen has resolution 1920×1080 and uses 8-bit RGB colour per channel, calculate the raw (uncompressed) size of one frame in megabytes.

Model Answer:

  • Pixels = 1920 × 1080 = 2,073,600 pixels.
  • Bytes per pixel = 3 channels × 1 byte = 3 bytes.
  • Total bytes = 2,073,600 × 3 = 6,220,800 bytes.
  • Convert to MB: 6,220,800 ÷ 1,048,576 ≈ 5.93 MB per frame.
  • At 60 fps this would be ~355 MB/s of raw data — showing why compression is essential.
  • This calculation demonstrates why formats like JPEG and PNG exist.

L02 — Raster Images (Ch3)

MCQs

1. A raster image is best described as:

A) A mathematical description of curves and shapes

B) A 2D array of pixels, each storing colour values

C) A 3D model projected onto a plane

D) A compressed audio-visual format

Answer: B — A raster image is fundamentally a 2D array (grid) of pixels, each containing colour information.

2. What is the key advantage of a vector image over a raster image?

A) Vector images load faster

B) Vector images are resolution independent

C) Vector images support more colours

D) Vector images are always smaller in file size

Answer: B — Vector images describe shapes mathematically, so they can be rendered at any resolution without quality loss.

3. In the textbook's 2D coordinate convention, where is the origin (0,0) of a raster image?

A) Top-left corner

B) Centre of the image

C) Bottom-left corner

D) Top-right corner

Answer: C — The textbook places the origin at the bottom-left, with i increasing right and j increasing upward.

4. What is "quantisation" (banding) in raster images?

A) Clipping pixel values above the maximum intensity

B) The colour-jumping artefact caused by rounding pixel values to fewer bits

C) Scaling the image to a lower resolution

D) Converting from RGB to greyscale

Answer: B — Quantisation occurs when storing images with fewer bits than the source, causing visible jumps between colour levels.

5. An 8-bit per channel RGB image can represent how many distinct values per channel?

A) 8

B) 64

C) 256

D) 1024

Answer: C — 2⁸ = 256 distinct values per channel (0–255).

6. Given a foreground cyan pixel (0, 1, 1) and a background yellow pixel (1, 1, 0) with alpha coverage α = 0.5, what is the blended result using c = αc_f + (1−α)c_b?

A) (0.5, 1.0, 0.5)

B) (0.0, 1.0, 0.0)

C) (1.0, 1.0, 1.0)

D) (0.5, 0.5, 0.5)

Answer: A — c = 0.5×(0,1,1) + 0.5×(1,1,0) = (0+0.5, 0.5+0.5, 0.5+0) = (0.5, 1.0, 0.5).

7. What does HDR (High Dynamic Range) imaging use to represent pixel values?

A) Integer values capped at 255

B) Floating-point numbers to represent values beyond [0,1]

C) 4-bit values for faster processing

D) Indexed colour palettes

Answer: B — HDR images use floating-point numbers to store values outside the standard 0–1 range, capturing brighter highlights and darker shadows.

8. Which file format is lossless?

A) JPEG

B) MP4

C) PNG

D) WebP (lossy mode)

Answer: C — PNG uses lossless compression; no image data is discarded.

9. You are developing a medical imaging application and need to store X-ray scans with no quality loss. Which compression type should you use?

A) Lossy compression (JPEG) to save disk space

B) Lossless compression (PNG/TIFF) to preserve all data

C) No compression; raw only

D) Vector format for compactness

Answer: B — Medical images must not lose diagnostic detail; lossless compression preserves every pixel value exactly.

10. Alpha compositing is described as the "over" operation. If the foreground is fully transparent (α = 0), what is the result?

A) The foreground colour entirely replaces the background

B) The result equals the background colour unchanged

C) The result is black

D) The result is the average of foreground and background

Answer: B — When α = 0: c = 0×c_f + 1×c_b = c_b — background is unchanged.

Long Questions

Q1. Compare Low Dynamic Range (LDR) and High Dynamic Range (HDR) images. Describe how each stores pixel values, the limitations of LDR, and a scenario where HDR is necessary.

Model Answer:

  • LDR images store pixel values as integers in [0, 255] per channel (8-bit), mapping the full visible range to 256 levels.
  • HDR images use floating-point representation, allowing values beyond 1.0 to capture brighter intensities (e.g., direct sunlight vs shadow in the same shot).
  • LDR limitation: clipping — any value above 255 gets clamped, losing highlight detail; quantisation causes banding in gradients.
  • HDR is necessary for: tone-mapped physically-based rendering, photography with extreme contrast (outdoor shots through a window), and lighting calculations that require absolute energy values.
  • HDR requires tone-mapping to display on standard LDR screens.

Q2. A game engine stores textures as JPEG to reduce file size. During runtime the artist notices visible colour banding in smooth gradients. Explain what causes this and what they should do instead.

Model Answer:

  • JPEG is a lossy format — it discards high-frequency colour information to reduce file size, which causes posterisation/banding in smooth gradients.
  • The 8-bit integer quantisation further limits colour precision, and JPEG's block-based compression introduces additional artefacts.
  • Solution: store textures as PNG (lossless) or use DXT/BC texture compression (GPU-native lossless for colour, designed for real-time).
  • For runtime: ensure mipmaps are generated to prevent aliasing at distance.
  • For gradients specifically, consider using 16-bit per channel to increase the number of representable colour levels.

Q3. A foreground pixel has colour (0, 1, 0) (green) and covers 30% of the background pixel (1, 0, 0) (red). Using alpha compositing c = αc_f + (1−α)c_b, compute the final colour. Then calculate the total number of possible RGBA pixel values for a 4-channel 8-bit image.

Model Answer:

  • α = 0.30, c_f = (0,1,0), c_b = (1,0,0)
  • c = 0.3×(0,1,0) + 0.7×(1,0,0) = (0+0.7, 0.3+0, 0+0) = (0.7, 0.3, 0.0)
  • For RGBA 8-bit: each channel has 2⁸ = 256 values; 4 channels total.
  • Possible combinations = 256⁴ = 2³² = 4,294,967,296 distinct RGBA values.
  • This is approximately 4.3 billion unique colours including transparency values.

L03 — Ray Tracing (Ch4)

MCQs

1. In the basic ray tracing algorithm, what is a "viewing ray"?

A) A ray from the light source to the screen

B) A ray from the viewpoint through a pixel into the scene

C) A ray bounced off a mirror surface

D) A ray used to calculate ambient light

Answer: B — A viewing ray originates at the viewpoint (camera) and passes through a pixel towards the scene.

2. What are the three core steps of the basic ray tracing algorithm?

A) Clipping, rasterisation, blending

B) Ray generation, ray intersection, shading

C) Vertex processing, fragment processing, depth testing

D) Modelling, texturing, lighting

Answer: B — Ray tracing: (1) generate viewing ray, (2) find closest object intersection, (3) compute shading at hit point.

3. In ray-sphere intersection, the equation (d·d)t² + 2d·(e−c)t + (e−c)·(e−c) − R² = 0 yields two solutions t₁ and t₂. What does it mean when the discriminant is negative?

A) The ray grazes the surface of the sphere

B) The ray intersects the sphere at two points

C) The ray does not intersect the sphere

D) The ray is inside the sphere

Answer: C — A negative discriminant means no real solutions exist, so the ray misses the sphere entirely.

4. For a ray-triangle intersection using barycentric coordinates, which conditions must β and γ satisfy for the intersection to be inside the triangle?

A) β < 0, γ < 0

B) β > 0, γ > 0, and β + γ < 1

C) β = 0, γ = 0

D) β > 1, γ > 1

Answer: B — The point is inside the triangle when β > 0, γ > 0, and their sum is less than 1 (ensuring α = 1−β−γ > 0 too).

5. What is the purpose of a shadow ray in ray tracing?

A) To determine the reflection direction

B) To determine if a light source is occluded at a surface point

C) To calculate ambient lighting

D) To anti-alias the rendered image

Answer: B — A shadow ray is cast from the hit point towards the light source; if it intersects another object first, the point is in shadow.

6. In perspective projection using ray tracing, what is special about the "focal length" d?

A) It controls the brightness of the scene

B) It is the distance from the eye to the image plane, determining the field of view

C) It sets the maximum ray bounce depth

D) It defines the size of the shadow rays

Answer: B — The plane distance d (focal length) determines how far the image plane is from the eye, directly affecting the field of view.

7. Compared to orthographic projection, perspective projection:

A) Produces views where all projection lines are parallel

B) Produces views where distant objects appear smaller, matching natural human perception

C) Is only used for 2D drawings

D) Does not involve a viewpoint position

Answer: B — Perspective projection converges rays at a single viewpoint, causing size reduction with distance — mimicking how humans see.

8. A ray tracer needs to find the closest intersection among 1 million triangles for every pixel at 1080p resolution. What data structure would most improve performance?

A) A linked list of all triangles

B) A Bounding Volume Hierarchy (BVH)

C) A 2D hash table

D) A sorted array of triangles by colour

Answer: B — A BVH spatially partitions objects so most scene geometry can be skipped per ray, reducing intersection tests from O(n) to O(log n).

9. Mirror reflection in ray tracing computes a new ray direction as r = d − 2(d·n)n. What do d and n represent?

A) d is the shadow ray; n is the light direction

B) d is the incoming ray direction; n is the surface normal at the hit point

C) d is the reflected colour; n is the diffuse coefficient

D) d is the normal; n is the viewing direction

Answer: B — d is the direction of the incoming (viewing) ray, and n is the unit surface normal at the intersection point.

10. An artist wants to render a glass sphere with both reflections and refractions using ray tracing. Which ray types must be generated at each intersection?

A) Only shadow rays

B) Shadow rays, reflected rays, and transmitted (refracted) rays

C) Only reflected rays

D) Only viewing rays

Answer: B — A glass surface requires: shadow ray (is the point lit?), reflected ray (mirror-like reflection), and refracted/transmitted ray (light bending through the glass).

Long Questions

Q1. Explain the difference between parallel (orthographic) and perspective projection. Describe when each type is used in practice and what properties are preserved in each.

Model Answer:

  • Orthographic projection: maps 3D points by moving them parallel to a projection direction onto the image plane; all projection lines are parallel.
  • Preserves: size, shape, parallel lines remain parallel — ideal for technical/engineering drawings where measurement accuracy matters.
  • Perspective projection: all rays originate from a single viewpoint (eye); objects further away appear smaller.
  • Preserves: visual realism matching the human eye; straight lines remain straight.
  • Orthographic use: CAD software, architectural plans, side-view game cameras.
  • Perspective use: 3D games, film VFX, any application requiring natural-looking 3D scenes.

Q2. A ray tracer renders a scene with a reflective metal ball on a wooden floor under a single point light. Describe step by step what rays are generated for one pixel that "sees" the ball, and what happens at each intersection.

Model Answer:

  • Step 1: Generate viewing ray from camera eye through the pixel into the scene.
  • Step 2: Find the closest intersection — the ray hits the metal ball surface.
  • Step 3: Cast a shadow ray from the hit point towards the point light; if unoccluded, apply diffuse + specular shading.
  • Step 4: Generate a reflected ray using r = d − 2(d·n)n; recursively trace this ray.
  • Step 5: The reflected ray hits the wooden floor — shade the floor with diffuse lighting + shadow test.
  • Step 6: Combine the ball's own shading colour + the reflected floor colour weighted by the mirror coefficient k_m.
  • Final pixel colour = ball_shading_colour + k_m × reflected_colour.

Q3. A ray with origin e = (0,0,0) and direction d = (0,0,−1) is tested against a sphere centred at c = (0,0,−5) with radius R = 1. Using the quadratic equation (d·d)t² + 2d·(e−c)t + (e−c)·(e−c) − R² = 0, find the intersection values of t.

Model Answer:

  • Compute (e−c) = (0,0,0) − (0,0,−5) = (0,0,5).
  • d·d = 0² + 0² + (−1)² = 1.
  • d·(e−c) = (0)(0) + (0)(0) + (−1)(5) = −5.
  • (e−c)·(e−c) = 0² + 0² + 5² = 25.
  • Quadratic: 1·t² + 2(−5)t + (25 − 1) = 0 → t² − 10t + 24 = 0.
  • Discriminant = 100 − 96 = 4 → √4 = 2.
  • t = (10 ± 2) / 2 → t₁ = 4, t₂ = 6.
  • The ray enters the sphere at t = 4 (front face) and exits at t = 6 (back face).

L04 — Surface Shading (Ch5)

MCQs

1. What is the Lambertian (diffuse) shading model based on?

A) The angle between the viewing direction and the reflected ray

B) The angle of incidence between the surface normal and the light direction

C) The distance from the camera to the surface

D) The colour of the background

Answer: B — Lambert's law: diffuse intensity is proportional to cos(θ) = n̂·l̂, where θ is the angle between normal and light direction.

2. In a point light illumination model, why does irradiance E decrease with the square of the distance (r²)?

A) Light photons lose energy over distance

B) The same power is spread over a sphere of surface area 4πr², so power per unit area decreases with r²

C) The shader applies a distance penalty manually

D) It is a constant approximation used for performance

Answer: B — Irradiance = I/r² because a point source emitting power P radiates it equally over a sphere; the sphere's surface area grows with r², so local intensity falls as 1/r².

3. What is a BRDF?

A) A ray tracing algorithm that finds the closest object

B) A function defining how light from a source is reflected off an opaque surface

C) A compression technique for texture maps

D) A type of anti-aliasing filter

Answer: B — BRDF (Bidirectional Reflectance Distribution Function) describes how much light is reflected in a given direction relative to an incoming direction.

4. Why is there no shadow computation for ambient light?

A) Ambient light is too dim to cast shadows

B) Ambient light has no single direction; it is a global approximation of indirect lighting

C) Shadow rays cannot intersect opaque objects

D) Ambient light only affects transparent surfaces

Answer: B — Ambient light represents the omnidirectional, indirect illumination in a scene — it has no specific source or direction, so no directional shadow test is possible.

5. In the Phong shading model, which term accounts for the shiny specular highlight?

A) k_a × L_a

B) k_d × L_d × max(n̂·l̂, 0)

C) k_s × L_s × max(v̂·r̂, 0)^α

D) k_m × shadeRay(...)

Answer: C — The specular term uses the reflection direction r̂ and viewing direction v̂; the exponent α controls highlight sharpness.

6. A surface has diffuse coefficient k_d = 0.8, the irradiance E = 1.0 W/m², and n̂·l̂ = 0.5. What is the diffuse colour contribution?

A) 0.4

B) 0.8

C) 1.0

D) 0.16

Answer: A — Diffuse = k_d × E × max(n̂·l̂, 0) = 0.8 × 1.0 × 0.5 = 0.4.

7. The Phong shading model has a "shininess" exponent α. What does increasing α do to the specular highlight?

A) Makes it larger and softer

B) Makes it smaller and sharper

C) Has no effect on the highlight shape

D) Changes the colour of the highlight

Answer: B — A higher α raises a value < 1 to a higher power, rapidly reducing it to zero — producing a tight, sharp specular spot.

8. Why must the dot product n̂·l̂ be clamped with max(0, n̂·l̂) in diffuse shading?

A) Negative values cause the GPU to crash

B) A negative value would mean the light is behind the surface and should contribute no light

C) To normalise the result to [0,1]

D) To prevent overbrightness at grazing angles

Answer: B — When the angle between normal and light exceeds 90°, the surface faces away from the light; clamping to 0 ensures back-lit surfaces don't receive negative illumination.

9. A scene has 3 point lights and 1 ambient light. How many times is the point light irradiance algorithm executed per shading point?

A) Once

B) Twice

C) Three times, one per point light

D) Four times

Answer: C — The point light shading algorithm is applied once per light source; results are summed. Ambient is computed once separately.

10. An artist sets k_d = (1,0,0) (red) and k_s = (1,1,1) (white) on a shiny material. Under a white light, the specular highlight will appear:

A) Red

B) Blue

C) White

D) Black

Answer: C — The specular highlight uses k_s (white) × light colour (white) = white. The diffuse component is red; the highlight itself is white (the light's colour).

Long Questions

Q1. Explain the three components of the Phong shading model (ambient, diffuse, specular). For each, state what physical phenomenon it approximates and when it dominates.

Model Answer:

  • Ambient (k_a × L_a): approximates indirect, omnidirectional bounce lighting from the environment — ensures no surface is completely black; dominates in occluded/shadow areas.
  • Diffuse (k_d × L_d × max(n̂·l̂, 0)): models Lambertian reflection — light scattered equally in all directions from rough surfaces; dominates on matte materials (clay, plaster) at surface normals facing the light.
  • Specular (k_s × L_s × max(v̂·r̂, 0)^α): models the shiny highlight from near-mirror-like micro-facet reflection; dominates on polished surfaces (plastic, metal) in the reflection direction.
  • Together: colour = ambient + Σ(diffuse_i + specular_i) for each light i.
  • The shininess exponent α separates material types: low α (matte), high α (glossy/metallic).

Q2. You are shading a metallic robot joint. Under a single point light the joint appears completely black even though it is facing the light. Diagnose at least three possible causes.

Model Answer:

  • The surface normal may be pointing away from the light, causing n̂·l̂ < 0 and the clamped diffuse term to be 0.
  • The shadow ray test may incorrectly detect self-occlusion (shadow acne) — the shadow ray intersects the object's own surface due to floating-point imprecision; fix with an epsilon offset.
  • k_d (diffuse coefficient) may be set to 0 — no diffuse contribution.
  • The light position may be on the wrong side of the surface due to a coordinate system error (world vs local space mismatch).
  • The irradiance calculation may use incorrect distance r — if r = 0 (degenerate), a divide-by-zero or very small r may produce NaN/infinity then clamped to zero.

Q3. A point light has intensity I = 100 W/sr and is located 5 metres from a surface. The angle between the surface normal and the light direction is 60°. The surface has k_d = 0.7. Calculate the diffuse colour contribution.

Model Answer:

  • Irradiance E = I / r² = 100 / 5² = 100 / 25 = 4.0 W/m².
  • n̂·l̂ = cos(60°) = 0.5.
  • Diffuse = k_d × E × max(n̂·l̂, 0) = 0.7 × 4.0 × 0.5 = 1.4 (arbitrary colour units scaled by material colour).
  • Note: in practice this would be scaled per channel (R, G, B) using the light's colour vector.
  • At r = 10m: E = 100/100 = 1.0 — showing the inverse-square falloff effect.

L05 — Graphical Hardware Introduction (Ch17)

MCQs

1. A GPU is optimised for which type of workload?

A) Sequential, branching computation with complex logic

B) Highly parallel computation across thousands of threads simultaneously

C) Large database queries

D) Network packet routing

Answer: B — GPUs contain thousands of smaller cores designed to execute the same operation on many data elements simultaneously (SIMT model).

2. OpenGL is best described as:

A) A programming language for writing games

B) A cross-platform API for interacting with graphics hardware, implemented as a state machine

C) A GPU driver written by NVIDIA

D) A shading language like GLSL

Answer: B — OpenGL is an API (not a language) that provides a cross-platform interface to graphics hardware, operating as a state machine where function calls change bound state.

3. What does the "state machine" metaphor mean for OpenGL?

A) OpenGL uses finite automata for rendering decisions

B) OpenGL maintains a set of current bindings and settings; draw calls use the currently bound state

C) OpenGL automatically switches between different rendering algorithms

D) OpenGL has a state machine for each shader program

Answer: B — In OpenGL, state (bound VAO, shader program, textures, etc.) persists until explicitly changed; rendering uses whatever is currently bound.

4. Which of the following is NOT a component of graphics hardware?

A) GPU cores

B) VRAM (video memory)

C) Instruction cache

D) Network interface card

Answer: D — A network interface card is not part of graphics hardware; GPU, VRAM, and instruction cache are all GPU components.

5. Why does the GPU have so many cores compared to a CPU?

A) GPUs are cheaper to manufacture with many cores

B) Graphics workloads (shading pixels) are embarrassingly parallel — the same shader runs independently on thousands of fragments at once

C) More cores increase clock frequency

D) To allow the GPU to run the operating system

Answer: B — Rendering tasks (per-vertex, per-fragment) are independent and uniform, making them ideal for the SIMT (Single Instruction Multiple Threads) GPU model.

6. In OpenGL, every function call starts with "gl". What does glClearColor(0,0,0,1) do?

A) Draws a black triangle

B) Sets the clear colour for the colour buffer to black (opaque)

C) Creates a new shader program

D) Binds a vertex buffer

Answer: B — glClearColor sets the colour used when glClear(GL_COLOR_BUFFER_BIT) is called, i.e. what colour the screen is filled with before drawing.

7. What is a Vertex Array Object (VAO) in OpenGL?

A) A texture map stored in GPU memory

B) An object that remembers the configuration of vertex attribute pointers and bound VBOs

C) A compiled shader program

D) A framebuffer for off-screen rendering

Answer: B — A VAO stores the vertex attribute configuration (which VBOs are bound, their layout), so you only set up the attributes once and then bind the VAO to use them.

8. A developer forgets to call glUseProgram(shaderID) before setting uniforms. What is likely to happen?

A) Uniforms are set on a default built-in shader

B) Uniforms are set on the currently active (previously bound) shader, not the intended one

C) OpenGL generates a crash

D) The uniforms are buffered and applied to the next shader bound

Answer: B — glUniform calls operate on the currently active shader program; forgetting glUseProgram means uniforms may go to the wrong (or no) program.

9. The NVIDIA GeForce 256 (1999) was marketed as significant because it was:

A) The first GPU to support ray tracing

B) The first single-chip processor with integrated transform, lighting, and rendering — the first "real GPU"

C) The first GPU to support OpenGL 4.6

D) The first GPU with more than 1 billion transistors

Answer: B — The GeForce 256 was the first chip to integrate T&L (transform and lighting) on the GPU itself, defining the modern GPU.

10. Beyond computer graphics, GPUs are widely used in which other field?

A) Text processing

B) Machine learning and neural network training, due to their massive parallelism

C) File system management

D) Compiler design

Answer: B — Deep learning training involves massively parallel matrix multiplications — exactly what GPUs are designed for, leading to their widespread use in AI.

Long Questions

Q1. Explain what it means for OpenGL to be a "state machine." Describe at least four types of state in OpenGL and what happens if state is incorrectly set before a draw call.

Model Answer:

  • OpenGL maintains a global set of state variables; each gl* call either changes state or issues a draw command using the current state.
  • Vertex state: bound VAO and VBOs — determines vertex data layout; wrong binding = wrong geometry.
  • Shader state: active shader program (glUseProgram) — determines vertex and fragment processing; no program = default or undefined behaviour.
  • Texture state: bound textures per texture unit — determines which image data is sampled; wrong binding = wrong texture applied.
  • Depth/blend state: glEnable(GL_DEPTH_TEST), glEnable(GL_BLEND) — determines depth testing and transparency; forgetting = no depth testing = incorrect occlusion.
  • Example mistake: binding VAO for object A, then calling glDrawArrays intending to draw object B — the wrong geometry is rendered.

Q2. A student's OpenGL scene renders correctly on their NVIDIA laptop but is completely black on a lab machine with an AMD GPU. List four potential causes specific to cross-platform/cross-vendor OpenGL issues.

Model Answer:

  • GLSL shader compatibility: some vendors accept non-standard GLSL syntax; strict AMD drivers reject it — check for specification-compliant GLSL.
  • Uninitialized uniforms: if a uniform like the MVP matrix is never uploaded (glUniformMatrix4fv), some drivers default to identity, others to zero — results differ per vendor.
  • Extension support: using vendor-specific extensions (e.g., GL_NV_*) unavailable on AMD.
  • Coordinate system/winding order: if backface culling is enabled and the winding order of geometry is wrong, the scene renders inside-out — may be masked by vendor differences in default state.
  • Driver version: OpenGL version support differs; ensure the requested version is available with GLFW window hints.

Q1. Compare the z-buffer algorithm and the Painter's Algorithm for hidden surface removal. Describe how each works and explain why the z-buffer is preferred in modern real-time rendering.

Model Answer:

  • Painter's Algorithm: sorts all primitives back-to-front by depth and renders them in that order; back objects are overdrawn by front objects.

Q1. Describe the complete coordinate space transformation pipeline from object space to screen space, naming each space, the matrix that creates the transition, and what that matrix represents physically.

Q1. Explain why perspective projection causes objects to appear smaller as they move further away. Derive this from the perspective equation y_screen = (n/z) × y_eye.

Model Answer:

  • In perspective, the image plane is at z = −n (near plane). A point at eye-space position (x, y, z) projects to y_screen = (n / |z|) × y.
  • As z increases (object moves further), n/z → 0, so y_screen → 0 — the projected height shrinks.
  • This matches human perception: parallel lines (train tracks) converge to a vanishing point at infinity.
  • The perspective matrix encodes this by putting z into w; the divide (y/w) = (y×n)/(z) produces the foreshortening.
  • Orthographic has no z in the denominator: y_screen = y — size is preserved regardless of depth.

Q2. A developer complains that their perspective scene has "z-fighting" (flickering between two coplanar surfaces). Explain the technical cause and propose two solutions.

Model Answer:

  • Z-fighting: two surfaces very close in depth get the same z-buffer value due to floating-point precision; which one "wins" the depth test flickers per-frame.
  • Root cause: perspective projection compresses z non-linearly (1/z distribution); near-far range too large wastes precision at small z differences.
  • Solution 1: Increase the near plane distance (e.g., from 0.01 to 0.1) — this restores precision in the region of interest; most precision loss happens when near ≈ 0.
  • Solution 2: Use a 32-bit depth buffer instead of 16-bit, providing more discrete levels.
  • Solution 3: Separate coplanar surfaces with a small geometry offset (polygon offset in OpenGL: glPolygonOffset).
  • The root principle: narrow the near-far ratio (f/n) as much as the scene allows.

Answer: B — Barycentric coordinates are a weighted combination of triangle vertices summing to 1; interior points have all weights in [0,1].

Long Questions

Q1. Explain why perspective-correct interpolation is necessary and what visual artefact appears without it. Use texture coordinates as a concrete example.

Model Answer:

  • When vertices are projected to screen space, depth information is compressed non-linearly by perspective division.
  • Naively interpolating texture UV coordinates linearly across screen-space triangle vertices ignores this compression.
  • Result: textures appear to "slide" or stretch — most visible on large perspective-angled polygons (like a floor receding into the distance).
  • The corridor/checkerboard floor test: without correction, tiles appear warped near the edges; with correction, they look uniform.
  • Fix: interpolate u/w and v/w (and 1/w) linearly (these are linear in screen space), then at each pixel compute u = (u/w) / (1/w).
  • Modern GPU hardware performs this automatically; it was a visible artefact on older software renderers (early PS1 games are a famous example).

Q2. Describe three strategies for culling geometry before it reaches the fragment shader, explaining when each is applied in the pipeline.

Model Answer:

  • Frustum culling (CPU/application level): test object/mesh bounding volumes against the view frustum before submitting draw calls — avoids sending invisible geometry to the GPU entirely.
  • Backface culling (rasteriser): GPU compares triangle winding order against the camera direction; back-facing triangles are discarded before fragment shading — halves fragment load on closed meshes.
  • Occlusion culling (application/hardware): for objects entirely hidden by closer geometry — use hardware occlusion queries or precomputed PVS to skip draw calls; applied at draw-call submission.
  • Early-Z (hardware): GPU tests the z-buffer before running the fragment shader; occluded fragments never execute shading — saves fragment shader cost.
  • Together these form a layered defence reducing unnecessary GPU work at progressively later pipeline stages.

Q3. Triangle vertices in screen space are A=(0,0), B=(4,0), C=(2,4). Point P=(2,1). Compute the barycentric coordinates of P using the area method: α = Area(PBC)/Area(ABC), β = Area(APC)/Area(ABC), γ = Area(ABP)/Area(ABC).

  • Area(PBC): P=(2,1), B=(4,0), C=(2,4). (B−P)=(2,−1), (C−P)=(0,3). Cross: 2×3−(−1)×0 = 6. Area = 6/2 = 3.
  • Area(APC): A=(0,0), P=(2,1), C=(2,4). (P−A)=(2,1), (C−A)=(2,4). Cross: 2×4−1×2 = 6. Area = 6/2 = 3.
  • Area(ABP): A=(0,0), B=(4,0), P=(2,1). (B−A)=(4,0), (P−A)=(2,1). Cross: 4×1−0×2 = 4. Area = 4/2 = 2.

  • Check: 0.375 + 0.375 + 0.25 = 1.0 ✓. All positive → P is inside the triangle ✓.

Long Questions

Q1. Describe the four properties desirable in a texture coordinate function (bijectivity, size distortion, shape distortion, continuity). Give an example of a real texture mapping problem where one of these properties is violated.

Model Answer:

  • Bijectivity: one-to-one mapping between surface and texture space. Violation: cube-map projection maps the top of a sphere to a single point (pole singularity) — multiple surface points share one texel.
  • Size distortion: texture scale approximately constant across the surface. Violation: a naive sphere UV map (latitude/longitude) heavily compresses texels near the poles — the same texture area covers much less surface there.
  • Shape distortion: small circles on texture appear as circles on the surface. Violation: wrapping a photo onto a sphere stretches faces at the edges.
  • Continuity: nearby surface points → nearby UV coordinates. Violation: a UV seam on a character's back — the two sides of the cut are far apart in UV space, potentially causing visible lighting discontinuities.
  • UV unwrapping in tools like Blender tries to minimise all four simultaneously.

Q2. A game character's sword needs to have a metal texture. The artist provides a 1024×1024 texture. Describe the complete pipeline from texture file to a shaded fragment colour, including CPU and GPU steps.

Model Answer:

  • CPU: load image with stb_image (returns pixel data array); call glGenTextures, glBindTexture(GL_TEXTURE_2D), glTexImage2D to upload to GPU; call glGenerateMipmap.
  • CPU: set UV coordinates per vertex in the sword's VBO; configure attribute pointer for the UV channel.
  • GPU vertex shader: pass UV through as out vec2 TexCoord (varying).
  • Rasteriser: interpolates UV across the triangle using barycentric (perspective-correct) interpolation.
  • GPU fragment shader: uniform sampler2D metalTexture; then vec4 texColor = texture(metalTexture, TexCoord);.
  • Combine with Phong lighting: vec3 result = texColor.rgb * diffuse + specular; → FragColor = vec4(result, 1.0).
  • The texel colour becomes the diffuse material colour (k_d) in the shading equation.

Q3. A texture is mapped to a floor as u = 0.1x, v = 0.1z. The floor runs from x=0 to x=20, z=0 to z=20. Calculate the UV range covered and explain whether the texture will tile. Then compute the UV coordinate for the point (15, 0, 10).

  • UV range: u from 0.1×0 to 0.1×20 = 0 to 2; v from 0 to 0.1×20 = 0 to 2.
  • The UV range [0,2] exceeds [0,1] — with repeat/wrap mode, the texture tiles 2× in each direction across the floor (4 total tiles).
  • Point (15,0,10): u = 0.1×15 = 1.5, v = 0.1×10 = 1.0.
  • With wrap mode: effective u = frac(1.5) = 0.5, effective v = frac(1.0) = 0.0 (or 1.0 — same boundary).
  • The texture lookup will sample the texel at approximately the centre-left of the texture image.

1. What is a mipmap?

A) A special type of normal map

B) A pre-generated sequence of progressively halved-resolution versions of a texture for use at different distances

C) A texture compressed with lossy compression

D) An environment map for reflections

Answer: B — Mipmaps (MIP = multum in parvo, "much in little") pre-compute the texture at 1/2, 1/4, 1/8... resolution; the GPU selects the appropriate level based on screen-space coverage.

2. Why do mipmaps reduce aliasing on distant surfaces?

A) They increase the texture resolution

B) The pre-filtered lower-resolution mip levels avoid sampling a high-res texture too sparsely, which would cause Moiré patterns

C) They apply anti-aliasing in screen space

D) They reduce the polygon count of the mesh

Answer: B — Without mipmaps, a distant surface samples the full-resolution texture at too few points per texel (undersampling), causing flickering/Moiré; lower mip levels pre-average the texels.

3. A bump map stores:

A) Colour information for diffuse shading

B) Perturbations to the surface normal to simulate small surface detail without geometry

C) Displacement offsets applied to vertex positions

D) Ambient occlusion values

Answer: B — A bump map (and its successor, normal map) encodes how to perturb the surface normal per-fragment, simulating the appearance of small bumps and grooves in lighting.

4. What is the key difference between a bump map and a displacement map?

A) Bump maps are 3-channel; displacement maps are 1-channel

B) Bump maps perturb the normal (fake geometry); displacement maps actually offset vertex positions (real geometry)

C) Displacement maps are cheaper than bump maps

D) They are identical

Answer: B — Bump/normal maps are per-fragment shading tricks; displacement maps tessellate and move actual geometry, affecting silhouettes and self-shadowing.

5. A normal map stores:

A) The height of surface bumps as a greyscale image

B) The perturbed surface normal at each texel, encoded as an RGB colour (XYZ → RGB)

C) Specular highlight positions

D) Vertex colours

Answer: B — Normal maps encode the surface normal vector (in tangent space) as RGB values: (0,0,1) → (0.5,0.5,1.0) flat blue — the characteristic blue tint of normal maps.

6. An environment map (cubemap) is used for:

A) Storing the scene's depth information

B) Approximating reflections of the surrounding environment on shiny surfaces

C) Applying shadows to the scene

D) Storing pre-computed global illumination

Answer: B — Environment maps (cubemaps, sphere maps) store the surrounding environment and are sampled using the reflection vector to approximate specular reflections on shiny surfaces.

7. In OpenGL, calling glGenerateMipmap after glTexImage2D:

A) Deletes the base texture and replaces it with a compressed version

B) Automatically generates all lower-resolution mip levels from the base texture

C) Uploads the texture to a specific mip level

D) Applies bilinear filtering to the texture

Answer: B — glGenerateMipmap automatically computes and stores all mip levels (level 1, 2, 3, ...) from the level 0 (base) texture.

8. Bilinear texture filtering interpolates:

A) Between two mip levels

B) Among the four nearest texels surrounding the sample UV coordinate, weighted by proximity

C) Along the mesh edges

D) Between the diffuse and specular components

Answer: B — Bilinear filtering takes the four nearest texels (2×2 neighbourhood) and linearly interpolates in both u and v dimensions, producing smooth texture sampling.

9. Trilinear filtering extends bilinear filtering by:

A) Using three texture channels

B) Also interpolating between two adjacent mip levels (bilinear within each mip + linear between mips)

C) Using three passes of the GPU pipeline

D) Applying an additional depth-based filter

Answer: B — Trilinear = bilinear filtering on the two closest mip levels + linear blend between them, eliminating the visible "mip seam" artefact.

10. A shiny metallic surface needs to reflect its surroundings. The reflection vector is computed as r = d − 2(d·n)n. What OpenGL texture technique uses r to look up the reflected environment?

A) Diffuse texture mapping

B) Environment mapping (cubemap lookup using the reflection vector)

C) Displacement mapping

D) Shadow mapping

Answer: B — Environment mapping samples a cubemap using the reflection vector: texture(environmentCubemap, reflectDir) in GLSL.

Long Questions

Q1. Explain mipmapping. Why is it necessary, how is it generated, and what artefact does it prevent? Describe what happens at the transition between mip levels.

Model Answer:

  • Problem: a distant surface might cover just 2×2 pixels on screen but the full texture is 1024×1024 — sampling 4 points from a 1M-texel texture misses most detail, causing Moiré patterns and aliasing.
  • Mipmap solution: pre-compute levels: level 0=1024×1024, level 1=512×512, ..., level 10=1×1. At render time, select the level whose texel size approximates the on-screen coverage.
  • Generation: repeatedly downsample by averaging 2×2 blocks (glGenerateMipmap does this automatically).
  • Artefact prevented: Moiré interference patterns and shimmering on distant textured surfaces.
  • Mip seam: abrupt transition between levels produces a visible sharp boundary in the texture; trilinear filtering blends between the two nearest levels, eliminating this seam.

Q2. A character model has a flat-shaded face that looks low-polygon. The artist wants to add skin pore and wrinkle detail without adding triangles. Which texture technique should they use and how does it work in the rendering pipeline?

Model Answer:

  • Use a normal map — a technique that perturbs per-fragment normals to simulate surface detail.
  • The normal map stores encoded surface normals (in tangent space) as an RGB texture; flat blue (0.5,0.5,1) means "no perturbation."
  • In the fragment shader: sample the normal map → decode from [0,1] to [−1,1] → transform from tangent space to world/view space using the TBN matrix (Tangent, Bitangent, Normal).
  • Use the perturbed normal in the Phong/PBR lighting calculation — areas with "bumps" encoded in the map catch light differently, visually appearing 3D.
  • Advantage: effectively free in polygon count — all the detail is a shading trick. Limitation: silhouette edges still look flat (no actual geometry change).

Q3. A texture is 256×256 and a surface covers 4×4 pixels on screen. Calculate which mip level should be selected. If the surface covers 1×1 pixel instead, which level is used? (Level 0 = 256×256)

Model Answer:

  • Mip level selection: find the level where texture resolution ≈ screen coverage.
  • Level n has resolution 256/2ⁿ × 256/2ⁿ.
  • Surface covering 4×4 pixels: we want 256/2ⁿ ≈ 4 → 2ⁿ = 64 → n = 6 (4×4 mip level). Level 6 = 4×4 resolution.
  • Surface covering 1×1 pixel: 256/2ⁿ ≈ 1 → 2ⁿ = 256 → n = 8 (1×1 mip level).
  • Formula: LOD = log₂(256 / coverage_pixels) = log₂(256/4) = log₂(64) = 6 and log₂(256/1) = 8.
  • Trilinear filtering would blend: for 4×4 coverage, blend between level 5 (8×8) and level 6 (4×4).

Long Questions

Q1. Compare forward rendering and deferred rendering. For each, describe how lighting is computed, and state the conditions under which each is preferred.

Model Answer:

  • Forward rendering: for each object, compute the contribution of all lights per fragment in a single pass. Cost = O(objects × lights × fragments). Simple to implement; works with transparency and MSAA naturally.
  • Preferred: few lights (< 10), simple scenes, mobile/browser, transparent objects.
  • Deferred rendering: Pass 1 (geometry pass) writes surface data (normal, albedo, depth) to G-buffers. Pass 2 (lighting pass) computes all lights per screen pixel using G-buffer data. Cost = O(pixels × lights). Handles hundreds of lights efficiently.
  • Preferred: many dynamic point lights (dungeon, city with street lamps), complex indoor scenes.
  • Trade-offs: deferred requires large G-buffers (memory cost), no MSAA by default, and a separate forward pass for transparent objects.

Q2. A mobile game is struggling with frame rate. The scene has many trees, a large terrain, and multiple characters. Describe four performance optimisations specific to graphics, with justification for each.

Model Answer:

  • LOD for trees: use high-poly meshes near the camera, switching to low-poly (< 50 triangles) at 50m+; mobile GPUs are vertex-limited with dense foliage geometry.
  • Frustum culling: trees outside the camera frustum are never submitted — large outdoor scenes often have < 20% of trees visible; massive draw call reduction.
  • Texture atlases: combine multiple small textures (leaves, bark, grass) into one texture atlas — reduces texture bind state changes and draw calls.
  • Reduced shader complexity: mobile fragment shaders should avoid per-pixel lighting for distant objects; use vertex shading (Gouraud) for far terrain.
  • Instanced rendering: the same tree mesh drawn at many positions using a single instanced draw call — avoids per-instance CPU draw call overhead.

Q3. A shadow map is rendered at 1024×1024. The scene is lit by a directional sun light casting shadows over a 100m×100m terrain. Calculate the shadow map texel density (texels per metre). If the terrain covers 2000×2000 texels of the shadow map, what problem arises and how is it addressed?

Model Answer:

  • Shadow map covers 100×100m at 1024×1024 texels.
  • Density = 1024 / 100 = 10.24 texels/metre.
  • Problem: a 2000×2000m area would require 2000 × (1024/100) ≈ 20,480 texels in one dimension — exceeding the 1024 shadow map by 20×. Result: shadow aliasing (blocky shadows, each texel covers 2m).
  • Solution 1: Cascaded Shadow Maps (CSM) — use multiple shadow maps for near/mid/far ranges; near cascade is small area (high density), far cascade covers the full terrain (lower density).
  • Solution 2: Percentage Closer Filtering (PCF) — sample multiple neighbouring shadow map texels and average, producing soft shadow edges (less visible aliasing).
  • Solution 3: Perspective shadow maps — warp the shadow map to concentrate texels where the camera sees close to the viewer.

7. What is a "vector field" visualisation challenge compared to scalar fields?

A) Vector fields have no spatial component

B) Vectors have both magnitude and direction — must visualise two quantities simultaneously; common techniques include arrow glyphs and streamlines

C) Vector fields cannot be rendered in real time

D) Vector fields are always 2D

Answer: B — Vectors have direction (e.g., wind direction) and magnitude (speed); visualising both requires careful design (arrow glyphs, colour for magnitude, streamlines for flow).

8. Streamlines in flow visualisation are:

A) Lines connecting vertices in a mesh

B) Curves that are everywhere tangent to the vector field, showing flow direction through the field

C) Shadow edges on a terrain

D) Scan lines used in rasterisation

Answer: B — Streamlines follow the vector field direction at each point, tracing the path a massless particle would take in the flow.

9. A medical CT scan produces a 3D volume of density values. A radiologist wants to see the bone structure. Which approach is most appropriate?

A) Render the mesh as a coloured point cloud

B) Set a transfer function that makes bone-density values opaque and soft tissue transparent, then use volume rendering

C) Render every voxel as a cube

D) Apply a Phong shader to every voxel

Answer: B — Transfer functions selectively make bones visible (high-density → opaque) while soft tissue is semi-transparent, allowing the radiologist to see through the body to the skeleton.

10. Why is colour map choice important in scalar field visualisation?

A) Some colour maps are faster to compute

B) Poorly chosen colour maps (e.g., rainbow) can introduce false perceptual boundaries and hide or exaggerate features in the data

C) Colour maps affect rendering performance

D) Colour maps define the isosurface threshold

Answer: B — The rainbow colour map creates perceptual discontinuities (false edges) and is non-monotonic in luminance; perceptually uniform maps (viridis, plasma) better represent data variations.

Long Questions

Q1. Compare isosurface rendering (Marching Cubes) and direct volume rendering. Describe the workflow of each, and explain when a scientist would choose one over the other.

Model Answer:

  • Isosurface (Marching Cubes): select a threshold value; march through the 3D grid; for each cube of 8 voxels, classify corners as inside/outside; look up the 256 possible edge intersection patterns; generate triangles at the boundary. Outputs a standard triangle mesh.
  • Choose when: a clear surface boundary exists (bone in CT, protein structure), standard mesh tools are needed, or hard-edged geometry is desired.
  • Direct volume rendering: define a transfer function mapping density → RGBA; cast rays through the volume; at each step, sample density, apply transfer function, composite opacity and colour.
  • Choose when: soft transitions matter (showing muscle AND bone simultaneously), exploratory analysis (easy to change what's visible by adjusting transfer function), or displaying fuzzy/graded phenomena like clouds or smoke.
  • Key trade-off: isosurfaces give crisp geometry; volume rendering gives fuller context but is harder to set up and more expensive.

Q2. A meteorologist wants to visualise a 3D wind field (velocity vectors at every grid point) over Europe. Describe two visualisation techniques that show flow direction and magnitude, and the strengths and weaknesses of each.

Model Answer:

  • Arrow/glyph glyphs: place arrows at sample points with direction matching the vector and length (or colour) encoding magnitude.
  • Strength: immediately shows local direction and speed; intuitive; works at any density.
  • Weakness: dense placement creates visual clutter; hard to trace flow paths.
  • Streamlines: integrate paths tangent to the vector field from seed points; colour-code by speed.
  • Strength: shows global flow structure and paths; reveals vortices and fronts.
  • Weakness: choice of seed points affects what is shown; static (don't show unsteady flow).
  • Animated particles (pathlines): release particles and animate them over time following the time-varying field.
  • Strength: intuitive temporal understanding; shows dynamic features like storms.
  • Weakness: more complex to implement; works only for time-varying data.

Q3. A volume dataset has 256×256×256 voxels with values in [0,1]. Isosurface extraction at value 0.5 is needed. Estimate the number of cubes processed by Marching Cubes, and explain what the 256 lookup table entries represent.

Model Answer:

  • Grid of voxels: 256×256×256 = 16,777,216 voxels.
  • Cubes (cells between voxels): 255×255×255 = 16,581,375 cubes.
  • Each cube has 8 corners; each corner is classified as 0 (below 0.5) or 1 (above 0.5) → 2⁸ = 256 possible configurations.
  • The 256 lookup table entries: each entry describes which edges of the cube are intersected by the isosurface (0–12 edges per cube) and which triangle configurations to generate at those intersections.
  • By symmetry (reflections/rotations), the 256 cases reduce to ~15 fundamentally distinct patterns.
  • For each cube, look up the configuration, interpolate edge intersection positions, and output 0–5 triangles.

A) Locking certain bones in a skeleton rig

B) The process where an animator provides data at key moments and the computer interpolates between them

Answer: B — Key-framing stores the scene state at important (key) frames; the computer fills in intermediate frames via interpolation.

2. What is the difference between "straight-ahead" and "pose-to-pose" animation?

A) They are identical techniques

B) Straight-ahead draws every frame sequentially; pose-to-pose plans key frames first, then fills in between

C) Straight-ahead is for digital animation; pose-to-pose for hand-drawn

D) Pose-to-pose requires motion capture

Answer: B — Straight-ahead: animate frame by frame in sequence (good for fluid motion). Pose-to-pose: set key poses first, then fill in-betweens (better for planned, structured animation).

3. "Squash and stretch" is a principle of animation that:

A) Uniformly scales objects

B) Exaggerates deformation to emphasise weight, elasticity, and impact of motion

C) Rotates characters around their centre of mass

D) Reduces polygon count for distant objects

Answer: B — Squash (compression on impact) and stretch (elongation in motion direction) makes animation feel alive and communicates material properties like bounciness.

4. Forward Kinematics (FK) in skeletal animation means:

A) The end-effector position is specified and joint angles are computed

B) Joint angles/rotations are specified from root to leaf; world positions are computed by multiplying down the hierarchy

C) Physics drives all joint rotations

D) Keyframes are automatically generated

Answer: B — FK: animator sets each joint's local rotation; world positions are computed by multiplying parent transforms down the hierarchy chain.

5. Inverse Kinematics (IK) is useful because:

A) It is faster than FK

B) The animator specifies the desired end-effector position and the system solves for joint angles automatically

C) It avoids gimbal lock

D) It only works for rigid bodies

Answer: B — IK: specify where the hand/foot should be → solver computes joint angles to reach that position. Natural for character foot placement on uneven terrain.

6. Rigid skinning attaches a skin vertex to:

A) All joints with weighted influence

B) A single fixed joint; the vertex moves exactly with that joint

C) The mesh bounding box

D) An interpolated position between two joints

Answer: B — Rigid skinning assigns each vertex to one joint — simple but causes "candy-wrapper" artefacts at joint bends where skin tears apart.

7. Smooth skinning uses the formula d = Σωᵢdᵢ. What does this compute?

A) The total joint rotation

B) The blended vertex displacement as a weighted sum of displacement vectors from multiple influencing joints

C) The average bone position

D) The interpolation between key frames

Answer: B — Smooth skinning: each vertex is influenced by multiple joints with weights ωᵢ summing to 1; the displacement is a weighted blend, producing smooth bending at joints.

8. What is "anticipation" in the 12 principles of animation?

A) Making the character react after an action

B) A preparatory motion before the main action, signalling to the viewer what is about to happen

C) Slowing down the animation near key poses

D) Looping an animation cycle

Answer: B — Anticipation: the character prepares for an action (winding back before throwing a punch) — makes actions feel intentional and gives the viewer time to follow.

9. Procedural animation generates motion using:

A) Hand-crafted key frames for every frame

B) Mathematical functions or simulations to automatically generate motion (e.g., a wave function for water)

C) Motion capture data from actors

D) Pre-rendered video sequences

Answer: B — Procedural animation uses equations/algorithms (wave functions, noise, physics solvers) to generate motion, avoiding manual key framing.

10. A character's arm needs to reach for a door handle on uneven terrain. Which animation technique automatically solves the joint angles for the shoulder-elbow-wrist chain to reach the handle?

A) Forward Kinematics

B) Blend shapes / Morph targets

C) Inverse Kinematics

D) Keyframe interpolation

Answer: C — IK: specify the target position of the hand (end-effector at the handle); the IK solver computes the shoulder/elbow/wrist rotations to reach it.

Long Questions

Q1. Compare Forward Kinematics (FK) and Inverse Kinematics (IK). Describe how each works algorithmically and give a practical example where each is the better choice.

Model Answer:

  • FK: animator specifies local rotation for each joint from root downward; world position = accumulated product of parent transforms × local transform (DFS down hierarchy).
  • Example: a waving arm gesture where the artist wants precise control over shoulder sway and elbow bend — FK gives full artistic control.
  • IK: animator specifies the desired world position of the end-effector; a solver (e.g., FABRIK, Jacobian inverse) computes joint angles to reach that target.
  • Example: a character walking on stairs — the feet must precisely contact each step; IK automatically adjusts hip-knee-ankle to place the foot correctly regardless of terrain.
  • FK is easier to implement but tedious for constrained tasks. IK is powerful for goal-directed motion but can have multiple solutions (ambiguity) requiring additional constraints.

Q2. A cartoon bouncing ball needs to convey weight and elasticity. Describe how three animation principles from Lasseter's 12 principles apply to this motion.

Model Answer:

  • Squash and stretch: as the ball hits the floor, squash it (compress vertically, spread horizontally); as it rises, stretch it (elongate in motion direction). A heavier ball squashes more; a rubber ball stretches more.
  • Anticipation: before a large bounce, slightly compress the ball before impact to signal the energy of the bounce.
  • Slow-in and slow-out: the ball decelerates as it reaches peak height (slow-out near apex) and accelerates as gravity pulls it down (slow-in near ground) — matching physics and making the motion feel natural.
  • Timing: the speed of the animation (frames between poses) communicates weight; a heavy ball falls fast and barely bounces; a light ball has slower, higher bounces.

Q3. A smooth-skinned character has vertex V influenced by two joints: J1 with weight ω1=0.7 producing displacement d1=(2,0,0), and J2 with weight ω2=0.3 producing displacement d2=(0,3,0). Using d = Σωᵢdᵢ, compute the final displacement of vertex V.

Model Answer:

  • d = ω1×d1 + ω2×d2
  • d = 0.7×(2,0,0) + 0.3×(0,3,0)
  • d = (1.4, 0, 0) + (0, 0.9, 0)
  • d = (1.4, 0.9, 0)
  • Check weights: ω1 + ω2 = 0.7 + 0.3 = 1.0 ✓ (weights must sum to 1 for correct blending).
  • The vertex moves more in the X direction (dominated by J1's 0.7 weight) and partially in Y (from J2).
  • If only rigid skinning to J1 were used: d = (2,0,0). Smooth skinning produces a more natural blend between joint influences.

L20 — Visual Perception (Ch19)

MCQs

1. Weber's Law states that the just-noticeable difference (JND) in stimulus intensity is:

A) Constant regardless of the base intensity

B) Proportional to the base stimulus intensity: ΔI/I = constant

C) Inversely proportional to the base intensity

D) Equal to the square root of the intensity

Answer: B — Weber's Law: ΔI = k×I; the threshold for detecting a change is a constant fraction of the current stimulus level (e.g., you notice 1% brightness change regardless of absolute brightness).

2. The human visual system has approximately how many types of cone cells, and what do they detect?

A) Two types: bright and dim

B) Three types: sensitive to short (blue), medium (green), and long (red) wavelengths

C) Four types: RGBY

D) One type — cones detect all colours equally

Answer: B — The retina has three cone types (S, M, L) tuned to short (blue ~420nm), medium (green ~530nm), and long (red ~560nm) wavelengths — the basis of trichromatic colour vision.

3. "Colour constancy" refers to:

A) Using a fixed colour palette in art

B) The perceptual ability to perceive an object's colour as approximately constant despite changes in illumination

C) Ensuring monitors display identical colours

D) Keeping RGB values within [0,1]

Answer: B — Colour constancy: a red apple appears red both in sunlight and fluorescent light — the visual system compensates for the illuminant's spectral power.

4. Which monocular depth cue involves closer objects appearing larger on the retina?

A) Stereopsis

B) Motion parallax

C) Relative size

D) Convergence

Answer: C — Relative size: familiar objects that appear smaller are judged as further away. This is a powerful monocular depth cue used in perspective drawing.

5. What is "contrast sensitivity"?

A) The maximum brightness a display can produce

B) The ability of the visual system to detect differences in luminance between adjacent regions; varies with spatial frequency

C) The colour accuracy of a monitor

D) The rate at which images can be perceived (fps)

Answer: B — Contrast sensitivity: how small a luminance difference can be detected. It peaks at mid spatial frequencies (~3–5 cycles/degree) and falls at high frequencies (fine detail threshold).

6. Motion blur in computer graphics simulates:

A) The GPU's rendering speed limitation

B) The eye's retinal persistence that smears fast-moving objects across adjacent frames

C) Depth of field blur for unfocused objects

D) Anti-aliasing of geometric edges

Answer: B — Motion blur arises because the eye and film/sensor integrate light over time; fast-moving objects leave a smear. CG motion blur simulates this for more natural-looking animation.

7. Stereopsis (binocular depth perception) requires:

A) Only one eye

B) Two horizontally separated viewpoints (as in the two human eyes), creating horizontal disparity between left and right retinal images

C) High display refresh rate

D) A fisheye lens

Answer: B — Stereopsis uses the horizontal offset between the left and right eyes to compute depth from the disparity between corresponding points in each eye's image.

8. The "Mach band" effect illustrates that:

A) The visual system accurately represents all luminance differences

B) The visual system enhances perceived contrast at luminance boundaries — edges appear sharper than the actual luminance gradient

C) The eye cannot perceive colour differences

D) Screen resolution determines perceived sharpness

Answer: B — Mach bands: lateral inhibition in the retina exaggerates luminance edges; light bands appear lighter and dark bands appear darker near the boundary than they actually are.

9. Why does depth of field (DOF) blur in CG improve realism?

A) It reduces polygon count

B) It matches the human eye's (and physical camera's) behaviour of blurring objects outside the focal plane, providing a strong depth cue

C) It reduces rendering time

D) It eliminates aliasing

Answer: B — DOF blur is a natural physical property of lenses and the eye; CG scenes without DOF look flat and "computer-generated." Adding DOF provides a strong depth cue.

10. The visual system is more sensitive to changes in which property at low light levels?

A) Colour hue

B) Luminance/brightness (detected by rod cells, which are achromatic)

C) Saturation

D) Texture detail

Answer: B — Rods (used in low-light/scotopic vision) are more numerous and sensitive to luminance but cannot detect colour; hence poor colour discrimination in dark conditions.

Long Questions

Q1. Explain how the Contrast Sensitivity Function (CSF) should inform decisions in computer graphics rendering, texture design, and display calibration.

Model Answer:

  • CSF shows human sensitivity peaks at ~3–5 cycles/degree of visual angle and drops at very high frequencies (fine detail beyond ~30 cycles/degree is invisible) and very low frequencies.
  • Texture design: no need for detail finer than the CSF's high-frequency cutoff at the expected viewing distance — saves texture resolution and GPU bandwidth.
  • Rendering: anti-aliasing at frequencies beyond the CSF cutoff is wasted computation; MSAA can use fewer samples for fine geometry aliasing that the eye cannot resolve.
  • Compression: JPEG exploits CSF by discarding high-frequency components that the eye is less sensitive to (more compression in fine detail).
  • Display calibration: gamma correction aligns with the non-linear luminance response of the visual system; a perceptually uniform ramp requires gamma ≈ 2.2.
  • LOD: the CSF justifies switching to lower-detail meshes/textures at sufficient distance.

Q2. A VR experience wants to be convincing. Describe four visual perception phenomena that the VR system must correctly simulate to maximise immersion and depth perception.

Model Answer:

  • Stereopsis: the two displays must show images from the correct left/right eye positions with proper horizontal disparity — the most powerful depth cue in VR.
  • Motion parallax: as the user moves their head, the scene must update correctly at low latency (<20ms) — objects close by shift more than distant ones. High latency causes nausea (vestibular-visual conflict).
  • Vergence-accommodation conflict: in real life, eyes converge AND accommodate (focus) to the same depth; VR screens are at fixed distance, causing conflict — the most common cause of discomfort.
  • Depth of field: blurring objects not at the fixation depth matches natural vision and provides a depth cue; gaze-contingent DOF improves realism.
  • Colour constancy: consistent illumination across the scene prevents wrong colour percepts.

Q3. Weber's Law states ΔI/I = k where k ≈ 0.02 for luminance. A surface has luminance I = 100 cd/m². What is the smallest luminance change ΔI the eye can detect? If the luminance increases to 500 cd/m², what is the new ΔI?

Model Answer:

  • At I = 100 cd/m²: ΔI = k × I = 0.02 × 100 = 2 cd/m². Any change smaller than 2 cd/m² is imperceptible.
  • At I = 500 cd/m²: ΔI = 0.02 × 500 = 10 cd/m². The detection threshold increases proportionally.
  • Implication for CG: at high luminance (bright sunlight), larger intensity differences are needed to create perceivable contrast — validates using HDR for bright scenes.
  • Implication for quantisation: 8-bit (256 levels) is perceptually adequate because Weber's Law means we need the same relative precision, not absolute; equal steps in linear luminance are more than sufficient.

MCQs

1. The rendering equation describes:

A) How to transform vertices through the pipeline

B) The equilibrium of light energy at a surface: outgoing radiance equals emitted radiance plus reflected radiance integrated over the hemisphere

C) The equations for rigid body physics simulation

D) The formula for shadow map depth comparison

Answer: B — The rendering equation (Kajiya 1986) is L_o(x,ω_o) = L_e(x,ω_o) + ∫ f_r(x,ω_i,ω_o) L_i(x,ω_i) (ω_i·n) dω_i — the foundational equation for physically-based rendering.

2. A BRDF (Bidirectional Reflectance Distribution Function) describes:

A) The depth of shadows on a surface

B) The ratio of reflected radiance in a given outgoing direction to the incident irradiance from a given incoming direction

C) How light refracts through transparent materials

D) The speed of light in a medium

Answer: B — The BRDF f_r(ω_i, ω_o) defines how much light arriving from direction ω_i is reflected toward direction ω_o — the core of all PBR material models.

3. What does "energy conservation" mean for a physically-based BRDF?

A) The GPU uses less power

B) The total reflected light cannot exceed the total incident light — a surface cannot reflect more light than it receives

C) Light energy is converted to heat in the shader

D) Specular and diffuse must sum to exactly 1.0

Answer: B — A physically valid BRDF must be energy-conserving: ∫ f_r(ω_i,ω_o) cos(θ_i) dω_i ≤ 1 for all ω_o.

4. Radiometry is the science of:

A) Measuring electromagnetic radiation emitted by radio transmitters

B) Measuring the physical quantities of electromagnetic radiation (energy, power, radiance, irradiance) in absolute units

C) Rendering radio-wave phenomena

D) Measuring the refresh rate of displays

Answer: B — Radiometry provides the physical foundation for PBR: quantities like radiant flux (W), irradiance (W/m²), and radiance (W/m²/sr) describe light transport precisely.

5. The Fresnel effect describes:

A) Light scattering inside a material

B) The change in reflectance with the angle of incidence — surfaces reflect more at grazing angles

C) Light absorption in coloured materials

D) The blurring of reflections on rough surfaces

Answer: B — Fresnel equations describe how the fraction of light reflected vs transmitted changes with the angle of incidence; at grazing angles (~90°), most surfaces become mirror-like.

6. The Schlick approximation for Fresnel reflectance is:

A) R(θ) = R₀ + (1−R₀)(1−cosθ)⁵ where R₀ = ((n−1)/(n+1))²

B) R(θ) = sinθ/cosθ

C) R(θ) = R₀ × cosθ

D) R(θ) = 1 − (n−1)/(n+1)

Answer: A — Schlick's approximation is a fast, accurate approximation to the full Fresnel equations, widely used in real-time PBR shaders.

7. What is the "microfacet model" in PBR?

A) A model that approximates curved surfaces with small flat triangles

B) A surface model that treats a surface as covered with tiny mirror facets at the microscale; the distribution of these facets' orientations controls roughness

C) A model for rendering cloth at the fibre level

D) A GPU rendering model for mipmaps

Answer: B — Microfacet theory: real surfaces are microscopically rough; each facet is a perfect mirror; the distribution of facet normals (GGX, Beckmann) determines material roughness and specular shape.

8. In PBR, the "roughness" parameter controls:

A) The bump map intensity

B) The width of the specular highlight — low roughness = tight sharp specular; high roughness = broad diffuse-looking specular

C) The amount of ambient occlusion

D) The texture resolution

Answer: B — Roughness in microfacet PBR modulates the normal distribution function (NDF); low roughness = aligned facets → sharp specular; high roughness = varied facets → broad/diffuse specular.

9. Global illumination (GI) refers to:

A) Only direct lighting from light sources

B) The full simulation of inter-object light transport including indirect bounces (light from one surface illuminating another)

C) A post-process bloom effect

D) The ambient light term in the Phong model

Answer: B — GI encompasses all indirect light effects: diffuse interreflection, caustics, colour bleeding, and area soft shadows — everything beyond direct illumination.

10. Path tracing is a Monte Carlo method for solving the rendering equation. Its main disadvantage is:

A) It cannot simulate indirect lighting

B) It requires many samples per pixel to converge to a noise-free result — computationally very expensive

C) It only works for diffuse materials

D) It cannot handle specular reflections

Answer: B — Path tracing is unbiased and physically accurate but converges slowly; noise (fireflies) requires hundreds to thousands of samples per pixel for clean results.

Long Questions

Q1. Explain the rendering equation. Identify each term and explain what physical phenomenon it represents. Why is it difficult to solve in practice?

Model Answer:

  • L_o(x, ω_o) = L_e(x, ω_o) + ∫_Ω f_r(x, ω_i, ω_o) L_i(x, ω_i) (n̂·ω_i) dω_i
  • L_o: outgoing radiance at point x in direction ω_o — what the camera "sees."
  • L_e: emitted radiance — light emitted by the surface itself (e.g., a lamp).
  • f_r: BRDF — describes how the surface reflects light.
  • L_i: incoming radiance from direction ω_i — includes all light reaching x (direct + indirect).
  • n̂·ω_i: Lambert cosine factor — surface orientation relative to incoming light.
  • The integral: sum over the hemisphere of all incoming directions ω_i.
  • Why hard to solve: L_i itself contains the rendering equation recursively (indirect light = other surfaces' L_o); the integral over an infinite hemisphere has no closed form; Monte Carlo methods (path tracing) approximate it stochastically.

Q2. Explain how the Fresnel effect makes water look different when viewed from directly above versus at a shallow angle. Describe how Schlick's approximation is used in a PBR shader to implement this.

Model Answer:

  • Water has refractive index n ≈ 1.33. At normal incidence (looking straight down): R₀ = ((1.33−1)/(1.33+1))² = (0.33/2.33)² ≈ 0.02 — only 2% of light reflects; 98% refracts/transmits through. Water appears transparent.
  • At grazing incidence (looking along the surface): the Fresnel equations give R → 1.0 — nearly all light reflects. Water becomes mirror-like, reflecting the sky.
  • Schlick approximation in GLSL:

float fresnelSchlick(float cosTheta, float R0) {
    return R0 + (1.0 - R0) * pow(1.0 - cosTheta, 5.0);
}
float cosTheta = dot(viewDir, normal);
float F = fresnelSchlick(cosTheta, R0_water); // R0 ≈ 0.02
vec3 colour = mix(refractedColour, reflectedSkyColour, F);
  • At low angles (cosTheta → 0): F → 1 → see sky reflection. Overhead (cosTheta → 1): F → 0.02 → see through water.

Q3. A PBR material has R₀ = 0.04 (non-metal, like plastic). Using Schlick's approximation R(θ) = R₀ + (1−R₀)(1−cosθ)⁵, calculate the Fresnel reflectance at θ = 0° (normal incidence), θ = 60°, and θ = 90° (grazing). Interpret the results.

Model Answer:

  • θ = 0° (normal incidence): cosθ = 1. R = 0.04 + 0.96×(1−1)⁵ = 0.04 + 0 = 0.04 (4%). Most light passes through; small specular highlight.
  • θ = 60°: cosθ = 0.5. R = 0.04 + 0.96×(0.5)⁵ = 0.04 + 0.96×0.03125 = 0.04 + 0.03 = 0.07 (7%). Slightly more reflective.
  • θ = 90° (grazing): cosθ = 0. R = 0.04 + 0.96×(1)⁵ = 0.04 + 0.96 = 1.0 (100%). Perfect mirror at grazing angle.
  • Interpretation: even a matte plastic surface becomes mirror-like at grazing angles — this is why floors appear reflective when you look across them at a shallow angle, and why PBR materials without Fresnel look flat and fake.
