Global Scenario

A camera observes the scene from $\mathbf{e} = [3,\;4,\;5]^T$ gazing toward the origin with world-up $\mathbf{u} = [0,\;1,\;0]^T$. The output resolution is $800 \times 600$. Questions throughout this paper may reference this scenario.

View volume l,r: $-2,\;2$
View volume b,t: $-1.5,\;1.5$
Near/far planes: $n=1,\;f=100$

Question 1 Multiple Choice [16 marks • 2 each]
Important: Selecting more than one answer, unless the question specifies otherwise, will result in no marks being awarded. Choose carefully.
1.1
The viewport transformation $M_{vp}$ takes coordinates from which space to which space?
  • AWorld space → camera space
  • BCamera space → NDC
  • CCanonical view volume (NDC) → screen (pixel) space
  • DObject space → world space
$M_{vp}$ maps the canonical $[-1,1]^2$ view volume to the pixel grid of $n_x \times n_y$ pixels. It is the last transform in the viewing pipeline.
1.2
Which of the following best describes occlusion culling?
  • ADiscarding triangles whose normals point away from the camera
  • BClipping geometry that lies outside the view frustum
  • CDiscarding objects that are entirely hidden behind other objects
  • DReducing polygon count based on distance from the camera
Occlusion culling removes objects that are wholly occluded by closer geometry and would never contribute to the final image. Backface culling is (A); frustum culling is (B); LOD is (D).
1.3
What is the default winding order that OpenGL considers as front-facing?
  • AClockwise (CW)
  • BCounter-clockwise (CCW)
  • CDetermined by the surface normal direction
  • DIt has no default; must always be set manually
By default, OpenGL defines front faces as those whose vertices appear in counter-clockwise order when viewed from the front (glFrontFace(GL_CCW) is the default state).
1.4
When applying a texture to the marble floor, the texture coordinate $(s,t)$ falls slightly outside the $[0,1]$ range. Which wrapping mode causes the texture to clamp to the nearest border pixel rather than repeating?
  • AGL_REPEAT
  • BGL_CLAMP_TO_EDGE
  • CGL_MIRRORED_REPEAT
  • DGL_CLAMP_TO_BORDER
GL_CLAMP_TO_EDGE clamps to the closest texel on the texture edge, preventing seam artifacts. GL_CLAMP_TO_BORDER returns a configurable border colour instead.
1.5
Normal maps primarily store:
  • AHeight displacement values for vertex offset
  • BPerturbed surface normal directions (encoded as RGB)
  • CShadow depth values for shadow mapping
  • DDiffuse albedo colours
A normal map stores per-texel normal vectors, usually in tangent space, encoded as RGB (each channel maps $[-1,1]$ to $[0,255]$). They do not displace geometry, unlike displacement maps.
1.6
Which of the following factors influence Level of Detail (LOD) selection in a real-time game engine? Consider:
(i) Distance of the object from the camera   (ii) Screen-space size of the object   (iii) Whether the object is partially occluded
  • Ai only
  • Bii only
  • Ci and ii
  • Di, ii, and iii
Modern LOD systems can consider all three: camera distance, projected screen area, and partial occlusion all affect how much detail an object needs to look correct.
1.7
The Fresnel equations describe:
  • AThe attenuation of light through participating media
  • BThe fraction of light reflected vs. transmitted at a material boundary as a function of incident angle
  • CThe diffraction pattern created by small surface features
  • DThe relationship between roughness and specular lobe width
Fresnel equations give reflectance $R(\theta,\lambda)$ as a function of incident angle $\theta$ and wavelength $\lambda$. At grazing angles ($\theta \to 90°$), all materials become fully reflective.
1.8
According to the course material, which 3D model format is recommended for use with ASSIMP?
  • A.obj (Wavefront)
  • B.fbx
  • C.gltf 2.0
  • D.dae (Collada)
The course recommends .gltf 2.0 — it is an open, well-maintained format that stores geometry, materials, and scene hierarchy. ASSIMP supports it fully.
Question 2 Conceptual Questions [12 marks]
2.1   True or False — Backface Culling on All Meshes  [2 marks]
Motivation required • No marks without one
▶

State whether the following statement is true or false and provide a motivation. No marks will be awarded if no motivation is given.

“Backface culling should always be enabled for every mesh in a scene to improve performance.”
▼ Model Answer

FALSE. Backface culling is only safe for closed (watertight) meshes where interior faces are never visible. For open meshes (like a single flat plane, a leaf, or a one-sided wall), the “back” face may be visible from certain camera angles. Culling it would produce missing geometry. The glass sphere in the scenario is closed, so culling is safe there; a single floor tile is not.

2.2   Bump Maps vs Displacement Maps  [3 marks]
Distinguish the two and explain the main limitation of each
▶

The marble floor uses bump mapping to simulate surface detail. Explain (a) how bump maps differ from displacement maps, and (b) state the main limitation of bump mapping that displacement mapping overcomes.

▼ Model Answer

(a) A bump map perturbs the surface normal at shading time using encoded height gradients; the underlying geometry is unchanged. A displacement map physically moves vertices based on stored height values, actually changing the mesh geometry (requires sufficient tessellation).

(b) Bump maps cannot affect the silhouette of the object — at the edge of the marble tile the outline remains perfectly flat even though the surface appears bumpy. Displacement mapping modifies real geometry so the silhouette correctly matches the perceived surface detail.

2.3   Depth Prepass  [3 marks]
Explain what it is and why it is used in games
▶

Explain what a depth prepass is in the context of a real-time game renderer, and describe the performance advantage it provides.

▼ Model Answer

A depth prepass renders the entire scene once with a minimal vertex-only shader (no fragment computations), filling the depth buffer but producing no colour output. In the subsequent full lighting pass, the depth test (set to GL_EQUAL or GL_LEQUAL) ensures that only the front-most fragment at each pixel executes the expensive fragment shader.

Advantage: eliminates overdraw — fragments that would be overwritten by a closer surface are discarded early, before the costly lighting/texture computations run. This is especially valuable in scenes with heavy fragment shader work (PBR materials, many lights).

2.5   Forward Kinematics vs Inverse Kinematics  [2 marks]
Define both; state when IK is preferred
▶

Define forward kinematics (FK) and inverse kinematics (IK) for skeletal animation. State one scenario where IK is preferred over FK.

▼ Model Answer

FK: joint angles/rotations are specified directly; the position of the end effector is computed from those angles down the hierarchy. The animator controls each bone rotation.

IK: the desired end-effector position is specified; the system solves backwards to determine the joint angles that achieve it (a numerical or analytical solve).

Preferred scenario: IK is preferred when the end effector must touch or be constrained to a surface — e.g., a character’s foot planting on uneven terrain. Manually keying foot rotations in FK to match arbitrary ground height is impractical.

Question 3 Viewing Transformation Matrices [12 marks]
3.1   Camera Matrix $M_{cam}$  [6 marks]
Derive the camera frame vectors then build the full 4×4 matrix
▶

Using the camera position $\mathbf{e} = [3,\;4,\;5]^T$, gaze direction toward the origin, and world-up $\mathbf{u} = [0,\;1,\;0]^T$:

(a) Derive the camera frame vectors $\mathbf{w}$, $\mathbf{u}_{cam}$, and $\mathbf{v}$.

(b) Write the full $4 \times 4$ camera transform matrix $M_{cam}$.

▼ Model Answer

(a) Frame derivation:

Gaze vector $\mathbf{g} = \mathbf{origin} - \mathbf{e} = [0,0,0]^T - [3,4,5]^T = [-3,-4,-5]^T$

$\mathbf{w} = -\hat{\mathbf{g}} = \dfrac{[3,4,5]^T}{|[3,4,5]^T|}$,   $|[3,4,5]| = \sqrt{9+16+25} = \sqrt{50} = 5\sqrt{2}$

$$\mathbf{w} = \frac{1}{5\sqrt{2}}[3,4,5]^T \approx [0.424,\;0.566,\;0.707]^T$$

$\mathbf{u}_{cam} = \hat{\mathbf{u}} \times \mathbf{w}$ (where $\hat{\mathbf{u}} = [0,1,0]^T$):

$[0,1,0] \times \dfrac{1}{5\sqrt{2}}[3,4,5] = \dfrac{1}{5\sqrt{2}}[1\cdot5-0\cdot4,\;0\cdot3-0\cdot5,\;0\cdot4-1\cdot3] = \dfrac{1}{5\sqrt{2}}[5,\;0,\;-3]$

$|\mathbf{u}_{cam}| = \dfrac{\sqrt{25+0+9}}{5\sqrt{2}} = \dfrac{\sqrt{34}}{5\sqrt{2}}$, normalised: $\mathbf{u}_{cam} = \dfrac{[5,0,-3]^T}{\sqrt{34}} \approx [0.857,\;0,\;-0.514]^T$

$\mathbf{v} = \mathbf{w} \times \mathbf{u}_{cam}$ (cross product of the normalised vectors — left for student to verify).

(b) Camera matrix:

$$M_{cam} = \begin{bmatrix} u_x & u_y & u_z & -\mathbf{u}_{cam}\cdot\mathbf{e} \\ v_x & v_y & v_z & -\mathbf{v}\cdot\mathbf{e} \\ w_x & w_y & w_z & -\mathbf{w}\cdot\mathbf{e} \\ 0 & 0 & 0 & 1 \end{bmatrix}$$

The upper-left $3\times3$ is the camera frame as rows; the last column encodes the translation to move the world origin to the camera origin.

3.2   Orthographic Matrix $M_{orth}$  [3 marks]
Provide the fully-populated 4×4 matrix using the scenario parameters
▶

Using the view volume $[l,r,b,t,n,f] = [-2,\;2,\;-1.5,\;1.5,\;1,\;100]$, provide the complete $4\times4$ orthographic projection matrix $M_{orth}$.

▼ Model Answer
$$M_{orth} = \begin{bmatrix} \dfrac{2}{r-l} & 0 & 0 & -\dfrac{r+l}{r-l} \\[6pt] 0 & \dfrac{2}{t-b} & 0 & -\dfrac{t+b}{t-b} \\[6pt] 0 & 0 & \dfrac{2}{n-f} & -\dfrac{n+f}{n-f} \\[6pt] 0 & 0 & 0 & 1 \end{bmatrix}$$

Substituting values: $r-l=4$, $t-b=3$, $n-f=1-100=-99$, $r+l=0$, $t+b=0$, $n+f=101$:

$$M_{orth} = \begin{bmatrix} 0.5 & 0 & 0 & 0 \\ 0 & \tfrac{2}{3} & 0 & 0 \\ 0 & 0 & -\tfrac{2}{99} & -\tfrac{101}{99} \\ 0 & 0 & 0 & 1 \end{bmatrix}$$
3.3   Viewport Matrix $M_{vp}$  [3 marks]
Write the full 4×4 $M_{vp}$ for an 800×600 screen
▶

Write the $4\times4$ viewport transformation matrix $M_{vp}$ for a screen of $n_x = 800$ by $n_y = 600$ pixels.

▼ Model Answer
$$M_{vp} = \begin{bmatrix} \dfrac{n_x}{2} & 0 & 0 & \dfrac{n_x - 1}{2} \\[6pt] 0 & \dfrac{n_y}{2} & 0 & \dfrac{n_y - 1}{2} \\[6pt] 0 & 0 & 1 & 0 \\ 0 & 0 & 0 & 1 \end{bmatrix} = \begin{bmatrix} 400 & 0 & 0 & 399.5 \\ 0 & 300 & 0 & 299.5 \\ 0 & 0 & 1 & 0 \\ 0 & 0 & 0 & 1 \end{bmatrix}$$

This maps $x \in [-1,1]$ to pixel coordinates $[0,\;n_x-1]$ and $y \in [-1,1]$ to $[0,\;n_y-1]$, with the $-1$ offsets placing sample points at pixel centres.

4.2   Winding Order Question  [4 marks]
Determine winding and state whether culled
▶

A triangle has vertices (as they appear on screen, after projection):

$P_0 = (100,\;200)$, $P_1 = (300,\;50)$, $P_2 = (400,\;300)$ (all in screen pixels).

(a) Calculate the signed area using the formula $A = \tfrac{1}{2}[(x_1-x_0)(y_2-y_0)-(x_2-x_0)(y_1-y_0)]$ to determine winding order.

(b) If glFrontFace(GL_CCW) and glCullFace(GL_BACK) are active, will this triangle be culled?

▼ Model Answer

(a) $A = \tfrac{1}{2}[(300-100)(300-200)-(400-100)(50-200)]$
$= \tfrac{1}{2}[(200)(100)-(300)(-150)]$
$= \tfrac{1}{2}[20000 + 45000] = \tfrac{65000}{2} = 32500$

Since $A > 0$, the vertices are in counter-clockwise order (in screen space, where Y increases downward, a positive signed area = CCW).

(b) CCW is defined as front-facing. With GL_BACK culling, front-facing triangles are kept. This triangle will NOT be culled — it will be rendered.

Question 5 Texture Mapping & OpenGL Code [12 marks]
5.1   Texture Vertex & Fragment Shaders  [6 marks]
Complete GLSL shaders with texture coordinate pass-through
▶

Write a vertex shader and fragment shader pair for rendering the textured marble floor. Requirements:

  • Vertex shader inputs: vertex position at location = 0 (vec3) and texture coordinates at location = 1 (vec2).
  • Pass texture coordinates through to the fragment shader via an out/in variable.
  • Apply a uniform mat4 MVP transform to the position.
  • Fragment shader samples from a uniform sampler2D marbleTex and outputs the result as the fragment colour.
▼ Model Answer
// ── VERTEX SHADER ──
#version 330 core
layout(location = 0) in vec3 aPos;
layout(location = 1) in vec2 aTexCoord;

out vec2 TexCoord;

uniform mat4 MVP;

void main() {
    gl_Position = MVP * vec4(aPos, 1.0);
    TexCoord = aTexCoord;
}
// ── FRAGMENT SHADER ──
#version 330 core
in vec2 TexCoord;
out vec4 FragColor;

uniform sampler2D marbleTex;

void main() {
    FragColor = texture(marbleTex, TexCoord);
}
5.2   OpenGL Texture Setup  [4 marks]
C++ calls to create, bind, upload, and configure a texture
▶

Provide the C++ OpenGL calls to: (a) generate and bind a 2D texture object, (b) upload an RGB image loaded via stb_image into the texture (assume data, width, and height are available), and (c) enable mipmapping.

▼ Model Answer
// (a) Generate and bind
GLuint texID;
glGenTextures(1, &texID);
glBindTexture(GL_TEXTURE_2D, texID);

// Wrapping and filtering
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_REPEAT);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR_MIPMAP_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

// (b) Upload image data
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, width, height, 0,
             GL_RGB, GL_UNSIGNED_BYTE, data);

// (c) Generate mipmaps
glGenerateMipmap(GL_TEXTURE_2D);
5.3   Why glGenerateMipmap?  [2 marks]
Explain the visual artifact mipmapping prevents
▶

Explain why glGenerateMipmap is called after uploading a texture, and what visual artifact it prevents when the marble floor tile is viewed from a steep angle or at a distance.

▼ Model Answer

glGenerateMipmap pre-computes a chain of progressively smaller, filtered versions of the texture (each half the resolution of the previous). When a surface is far away or viewed at a steep angle, a single texture texel maps to many screen pixels — sampling the full-resolution texture causes texture aliasing (shimmering/Moiré patterns). The GPU selects the appropriate mipmap level (or blends between two) so that sampling is done at a resolution matched to the rendered size, eliminating this artifact.

Question 6 Physics-Based Rendering & Advanced Topics [8 marks]
6.1   Snell’s Law & the Glass Sphere  [4 marks]
Apply the law; classify the material
▶

A ray of light travelling through air ($n = 1.0$) strikes the glass sphere ($n_t = 1.5$) at an angle of incidence $\theta = 30°$.

(a) Apply Snell’s Law $n\sin\theta = n_t\sin\phi$ to find the transmission angle $\phi$.

(b) Is glass a dielectric or a conductor? Explain what this means for how it handles light.

▼ Model Answer

(a) $\sin\phi = \dfrac{n}{n_t}\sin\theta = \dfrac{1.0}{1.5} \sin 30° = \dfrac{1}{1.5} \times 0.5 = \dfrac{1}{3} \approx 0.333$

$\phi = \arcsin(0.333) \approx 19.47°$

(b) Glass is a dielectric (electrical insulator). Dielectrics transmit light through refraction as well as reflecting it. The fraction reflected vs. refracted is described by the Fresnel equations. At normal incidence, only a small fraction is reflected ($\approx 4\%$ for glass); at grazing angles, nearly all light is reflected (Fresnel effect).

6.2   Schlick’s Fresnel Approximation  [2 marks]
State the formula and explain the Fresnel effect
▶

State the Schlick approximation for the Fresnel reflectance $R(\theta)$, and explain qualitatively what the Fresnel effect looks like on the glass sphere from the scenario.

▼ Model Answer

Schlick approximation: $R(\theta) = R_0 + (1 - R_0)(1 - \cos\theta)^5$

where $R_0 = \left(\dfrac{n-n_t}{n+n_t}\right)^2$ is the normal-incidence reflectance.

Visual effect: when looking directly at the centre of the glass sphere (small $\theta$, near-normal incidence), only a small fraction of the spotlight is reflected — most passes through. At the rim of the sphere (large $\theta$, near-grazing), the sphere becomes highly reflective and you see a bright specular ring. This is the Fresnel effect.

6.3   ASSIMP Scene Loading  [2 marks]
Describe the aiScene tree structure
▶

When loading the glass sphere model using ASSIMP, describe the structure of the aiScene object returned by the importer, and identify the two main containers you would iterate over to extract vertex data and materials.

▼ Model Answer

The aiScene object is the root of a hierarchical scene graph:

  • mMeshes[] — array of aiMesh objects, each containing vertex positions, normals, texture coordinates, and face indices. Iterate this to extract geometry.
  • mMaterials[] — array of aiMaterial objects holding texture paths, colour properties (diffuse, specular, etc.). Mesh objects reference materials by index.
  • mRootNode — the scene hierarchy root (aiNode), where each node has a transformation matrix and indices into mMeshes[].

Process: traverse the node tree from mRootNode, accumulating transforms, and for each node, process the referenced meshes from mMeshes[] with their materials from mMaterials[].
